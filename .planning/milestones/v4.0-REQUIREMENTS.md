# Requirements -- v4.0 Brain API Control & CLI UI Ruling System

**Defined:** 2026-03-26
**Core Value:** Protect the moat (Brain API access control) and establish the visual grammar for all MindrianOS terminal interactions.

## v4.0 Requirements

### Brain API Control
- [x] **BRAIN-01**: Admin can create time-limited API keys for Brain access via CLI command
- [x] **BRAIN-02**: Admin can revoke, extend, and list active Brain keys
- [x] **BRAIN-03**: Supabase `brain_api_keys` table with expiry, usage tracking, plan tier
- [x] **BRAIN-04**: `brain_write` tool blocked for non-admin API keys
- [x] **BRAIN-05**: Email notification sent to admin when new access is requested
- [x] **BRAIN-06**: Supabase credentials wired into Render for production auth

### Admin Panel
- [x] **ADMIN-01**: `/mos:admin` command hidden from non-admin users (not locked -- invisible)
- [x] **ADMIN-02**: Admin panel self-teaches on every invocation (explains what each action does before executing)
- [x] **ADMIN-03**: Every destructive action shows consequences and requires confirmation

### CLI UI Ruling System
- [x] **UI-01**: `skills/ui-system/SKILL.md` auto-loaded on every session with full rendering rules
- [x] **UI-02**: 4-zone output anatomy enforced (header, body, intelligence strip, footer)
- [x] **UI-03**: 5 body shapes implemented (Mondrian board, semantic tree, room card, document view, action report)
- [x] **UI-04**: Symbol vocabulary (12 glyphs) and color contract (5 ANSI colors) codified
- [x] **UI-05**: Session start contract (cold/warm/signals) with max 2 signals in greeting
- [x] **UI-06**: Cross-surface adaptation rules (CLI -> Desktop -> Cowork)
- [x] **UI-07**: Dual context per folder -- STATE.md + MINTO.md both read before routing

### Multi-Room
- [x] **ROOM-01**: `.rooms/registry.json` with room index and active room tracking
- [x] **ROOM-02**: `/mos:rooms` command (list, new, open, close, archive, where)
- [x] **ROOM-03**: Context safety -- active room lock on all file-writing commands
- [x] **ROOM-04**: Header canary -- room name always visible in Zone 1
- [x] **ROOM-05**: Session start shows multi-room context when multiple rooms registered

### Autonomous Engine
- [x] **ACT-01**: `/mos:act` reads room STATE.md + MINTO.md, selects framework via Brain (local fallback)
- [x] **ACT-02**: Framework execution via subagent (`agents/framework-runner.md`, isolated context)
- [x] **ACT-03**: Thinking trace displayed before any action
- [x] **ACT-04**: `--chain` mode chains 3-5 frameworks, each feeds the next via output contract
- [x] **ACT-05**: `--dry-run` previews without executing

### Data Room Export v2
- [x] **EXPORT-01**: Self-contained HTML export with Mondrian grid overview
- [x] **EXPORT-02**: Document reader with sidebar navigation and TOC
- [x] **EXPORT-03**: Intelligence view (gaps, convergence, contradictions)
- [x] **EXPORT-04**: Cytoscape knowledge graph view
- [x] **EXPORT-05**: Rich content rendering (Mermaid, SVGs, tables, markdown)
