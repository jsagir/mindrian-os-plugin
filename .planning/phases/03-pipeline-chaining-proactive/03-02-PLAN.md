**`scripts/analyze-room`** -- bash script following compute-state pattern:
- Shebang: `#!/usr/bin/env bash`, set -euo pipefail
- Takes one argument: room directory path (default: ./room)
- If room dir does not exist, output "NO_ROOM" and exit 0
- Three analysis sections:

1. **Gap Detection** (`## Gaps`):
   - Define SECTIONS list matching the 8 Room sections
   - For each section: count .md files (excluding ROOM.md and STATE.md)
   - Empty section -> `GAP:STRUCTURAL:{section}:HIGH:Section is empty`
   - Single entry -> check if all entries share same methodology (grep frontmatter) -> `GAP:SEMANTIC:{section}:MEDIUM:Only analyzed via {methodology} -- consider additional perspectives`
   - Adjacent section gap: if problem-definition AND solution-design have entries but market-analysis is empty -> `GAP:ADJACENT:market-analysis:HIGH:Problem and solution explored but no market evidence yet`
   - Venture stage filtering: read STATE.md for venture stage. If Pre-Opportunity, suppress gaps for financial-model and legal-ip (not relevant yet). If Investment, elevate all empty sections to HIGH.

2. **Convergence Detection** (`## Convergence`):
   - Collect all .md files across room sections (excluding ROOM.md, STATE.md)
   - Extract key themes: grep for `## ` headers and `**bold**` text
   - Lowercase, extract words > 4 chars, count occurrences
   - Terms appearing in 3+ entries across DIFFERENT sections: `CONVERGE:{term}:{count}:MEDIUM:Appears in {count} entries across multiple sections`
   - Filter out common methodology terms (analysis, framework, venture, methodology, section, evidence, definition) to reduce noise

3. **Contradiction Detection** (`## Contradictions`):
   - This is the hardest to do deterministically. Keep it simple for bash:
   - Extract lines containing "B2B", "B2C", "enterprise", "consumer", "individual" from all entries
   - If conflicting customer type terms appear in different sections -> `CONTRADICT:{section1}:{section2}:MEDIUM:Customer type mismatch -- {term1} vs {term2}`
   - Note: Full semantic contradiction detection is Claude's job (room-proactive skill handles the nuanced cases). The script catches structural contradictions only.

- End with `## End`
- Make executable (chmod +x)

**`skills/room-proactive/SKILL.md`** -- proactive intelligence skill:
- Frontmatter: name=room-proactive, description="Proactive Data Room intelligence. Surfaces gaps, contradictions, and convergence signals. Active when room/ exists with entries."
- Title: "Room Proactive -- Gap, Contradiction, and Convergence Detection"
- **When to Activate** section:
  - SessionStart: Run lightweight analysis, surface at most 2 HIGH-confidence findings in greeting
  - /mindrian-os:status: Include all HIGH + MEDIUM findings in status output
  - /mindrian-os:room --insights: Full analysis including LOW confidence
  - NEVER interrupt a methodology session with proactive findings
- **Gap Detection** section: Beyond structural gaps (empty sections from compute-state), detect semantic gaps. Section has entries but all from same methodology (single-lens). Section has entries but none with evidence/validation. Adjacent sections filled but connecting section empty. Phrase gaps as opportunities with command suggestions: "Your market-analysis room has trend data but no customer evidence -- consider /mindrian-os:analyze-needs."
- **Contradiction Detection** section: Scan Room entries for incompatible claims across sections. Compare target customer descriptions, market size assertions, problem definitions, timing assumptions. Phrase as tensions, not errors: "Your Domain Explorer says 'enterprise IT' but your Lean Canvas targets 'individual developers' -- worth reconciling." IMPORTANT: Only flag contradictions between entries from similar time periods. Natural pivots (earlier X, recent Y) are progress, not contradictions.
- **Convergence Detection** section: Multiple frameworks reaching similar conclusions. Same domain in 3+ artifacts, same customer segment from different methodologies, same risk from multiple analyses. Phrase as signal strength: "Three frameworks independently identified 'aging water infrastructure' -- this convergence suggests a strong problem signal."
- **Confidence Scoring** table: HIGH (direct structural evidence, 3+ supporting entries, show in greeting), MEDIUM (2 supporting entries, keyword overlap, show in status), LOW (single entry inference, weak keyword match, show only on request).
- **Noise Gate** section: SessionStart MAX 2 findings. Prioritize 1 gap + 1 convergence (or contradiction). Never interrupt methodology sessions. If venture stage Pre-Opportunity, suppress financial-model and legal-ip gap alerts. If venture stage Investment, elevate gap alerts for empty sections.
- **Reading analyze-room Output** section: When analyze-room script output is available in session context, parse the structured lines (GAP:, CONVERGE:, CONTRADICT:) and use them as starting points for deeper Claude-driven analysis. The script catches structural patterns; you add semantic interpretation.
