1. `brain_framework_chain` -- Given current frameworks + problem_type, recommend next framework via FEEDS_INTO/TRANSFORMS_OUTPUT_TO/CO_OCCURS/ADDRESSES_PROBLEM_TYPE. LIMIT 5.
2. `brain_grade_calibrate` -- Match Example nodes with APPLIED_IN to get rubric_scores distribution, grade_numeric percentile. LIMIT 20.
3. `brain_find_patterns` -- Find similar ventures via Tool/Framework co-occurrence patterns. LIMIT 10.
4. `brain_concept_connect` -- GraphRAG: immediate connections from a concept node. LIMIT 20.
5. `brain_cross_domain` -- Cross-domain discovery between two domains via shared Framework/Concept connections. LIMIT 15.
6. `brain_contradiction_check` -- Find Framework pairs with conflicting ADDRESSES_PROBLEM_TYPE effectiveness for the same ProblemType.
7. `brain_gap_assess` -- Given current room frameworks (matched to Framework nodes), find missing PREREQUISITE and FEEDS_INTO targets. LIMIT 10.
8. `brain_search_semantic` -- Pinecone vector search pattern (not Cypher): call mcp__pinecone-brain__search-records with query text and optional metadata filters.

Use the Cypher examples from the research file. Every Cypher query MUST have a LIMIT clause. Include a "How to Use" section explaining that agents/skills read this file on demand and adapt templates with specific parameter values before calling mcp__neo4j-brain__read_neo4j_cypher.
