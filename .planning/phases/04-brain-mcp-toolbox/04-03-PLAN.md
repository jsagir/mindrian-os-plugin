Description: "Cross-domain pattern discovery -- what connects to your work?"
Setup: Read references/brain/query-patterns.md. Read room/STATE.md.
Flow: (1) Ask user what concept/domain they want to explore (or infer from room). (2) Call brain_concept_connect to get immediate connections. (3) Call brain_cross_domain if user mentions a second domain. (4) Surface surprising connections -- the "aha moment" command. Highlight unexpected links between fields. (5) For each connection, suggest a methodology command that could explore it further.
allowed-tools: Read, mcp__neo4j-brain__read_neo4j_cypher

**commands/compare-ventures.md** -- `/mindrian-os:compare-ventures`
Description: "Who else did something like this -- and what happened?"
Setup: Read references/brain/query-patterns.md. Read room/STATE.md.
Flow: (1) Extract venture description from room state. (2) Call brain_find_patterns with venture description and domain. (3) Call brain_search_semantic for semantic similarity across embeddings. (4) Present: similar ventures found, what worked, what failed, applicable lessons. (5) Do NOT expose student names or personal data -- aggregate patterns only.
allowed-tools: Read, mcp__neo4j-brain__read_neo4j_cypher, mcp__pinecone-brain__search-records

**commands/deep-grade.md** -- `/mindrian-os:deep-grade`
Description: "Calibrated venture assessment -- scored against 100+ real projects"
Setup: Check Brain is connected. This command delegates to the Grading Agent.

(4) Larry presents the results with encouragement and specific next steps. The Grading Agent does the scoring; Larry wraps it in teaching context.
allowed-tools: Read

allowed-tools: Read
