  <action>
**Create `scripts/git-ops`** -- a bash script following the exact pattern of `scripts/room-registry` (set -euo pipefail, WORK_DIR resolution, case statement for subcommands).

Subcommands:

1. **`is-enabled <room_path>`** -- Check if git is initialized for this room. Returns exit 0 + prints "true" if `<room_path>/.git/` exists OR if room_path is inside a git repo (check with `git -C "$ROOM_PATH" rev-parse --git-dir 2>/dev/null`). Returns exit 1 + prints "false" otherwise. This is THE guard function -- every other subcommand calls it first.

2. **`init <room_path>`** -- Run `git init` in room_path. Create `.gitignore` with: `.DS_Store`, `*.tmp`, `.proactive-intelligence.json` (internal state, not for git). Create `.gitattributes` with LFS patterns: `*.pdf filter=lfs diff=lfs merge=lfs -text`, `*.png filter=lfs ...`, `*.jpg`, `*.mp4`, `*.zip`, `*.pptx` -- but ONLY files over threshold (the .gitattributes just declares patterns; actual LFS tracking happens on commit). Print "initialized" on success.

3. **`commit <room_path> <file_path> [message]`** -- Guard: exit 0 silently if `is-enabled` returns false (NEVER error). Stage the specific file with `git -C "$ROOM_PATH" add "$FILE_PATH"`. If no message provided, auto-generate from file path: extract section name and filename to produce messages like "methodology: explore-domains", "meeting: 2026-03-30-investor-call", "manual: competitive-analysis/swot". Commit with `git -C "$ROOM_PATH" commit -m "$MSG" --no-verify 2>/dev/null || true` (--no-verify because these are auto-commits, not user commits; || true because empty commits should not fail). After commit, check auto_push setting and call push subcommand if "auto".

4. **`push <room_path>`** -- Guard: exit 0 silently if not enabled. Check if remote exists: `git -C "$ROOM_PATH" remote get-url origin 2>/dev/null`. If no remote, exit 0 silently. Push: `git -C "$ROOM_PATH" push origin HEAD 2>/dev/null &` (background, non-blocking). Print "pushed" if remote exists.

5. **`lfs-setup <room_path>`** -- Guard: exit 0 silently if not enabled. Check if `git lfs` is available (`git lfs version 2>/dev/null`). If available, run `git -C "$ROOM_PATH" lfs install --local`. If not available, print warning to stderr but do NOT fail: `echo "Git LFS not installed -- large files will be committed normally" >&2`. Exit 0 either way.

6. **`status <room_path>`** -- Guard: prints "git: disabled" and exits 0 if not enabled. Otherwise prints JSON: `{ "enabled": true, "remote": "<url or null>", "auto_push": "<setting>", "lfs": true/false, "dirty_files": N }`. Read auto_push from registry by finding the room name from path.

7. **`get-auto-push <room_path>`** -- Helper to read auto_push from registry. Find room name by matching path in registry. Default "off" if not found. Uses python3 one-liner (same pattern as room-registry).

Make the script executable (chmod +x).

**CRITICAL CONSTRAINT:** Every subcommand that performs a git operation MUST start with the is-enabled guard. If git is not enabled, the subcommand exits 0 (success, not failure). No error messages. No warnings. Silent no-op. This ensures rooms without git are NEVER affected.

**Create `lib/core/git-ops.cjs`** following the exact pattern of `lib/core/room-ops.cjs`:
- `'use strict'`, require child_process/fs/path, SCRIPTS_DIR resolved to ../../scripts
- Pure Node.js built-ins only, wraps bash scripts via execSync

Functions to export:

1. **`isGitEnabled(roomDir)`** -- Calls `scripts/git-ops is-enabled <roomDir>`. Returns boolean. Catches errors, returns false on any failure.

2. **`commitArtifact(roomDir, filePath, message)`** -- Calls `scripts/git-ops commit <roomDir> <filePath> [message]`. If message is omitted/null, lets the bash script auto-generate. Returns `{ committed: boolean, message: string }`. Catches errors, returns `{ committed: false, message: 'git not enabled' }` on failure. NEVER throws.

3. **`pushIfConfigured(roomDir)`** -- Calls `scripts/git-ops push <roomDir>`. Returns boolean (true if push attempted). NEVER throws.

4. **`initRepo(roomDir)`** -- Calls `scripts/git-ops init <roomDir>`. Returns boolean. NEVER throws.

5. **`setupLfs(roomDir)`** -- Calls `scripts/git-ops lfs-setup <roomDir>`. Returns boolean. NEVER throws.

6. **`getStatus(roomDir)`** -- Calls `scripts/git-ops status <roomDir>`. Returns parsed JSON or `{ enabled: false }` on failure. NEVER throws.

Every function wraps in try/catch. Every function returns a safe default on error. No function ever throws -- git operations are ENHANCEMENTS, not gates.
