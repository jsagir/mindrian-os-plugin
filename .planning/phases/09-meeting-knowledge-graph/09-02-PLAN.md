  <action>
Extend dashboard/index.html with the following additions. Read the full file first to understand the existing structure.

**New Cytoscape.js stylesheet rules** (add after existing node/edge styles):
- `node.meeting`: gold `#D4A843` diamond, 50x50px, label below
- `node.speaker`: deep blue `#1E3A6E` ellipse, 40x40px, label below
- `node.concept`: warm yellow `#C8A43C` rectangle, 30x30px, label below
- `node.concept.unresolved`: same as concept but with 2px red `#A63D2F` border, 0.5 background-opacity
- `edge.spoke-in`: light gray `#A09A90` arrow, width 1
- `edge.filed-to`: gray `#5C5A56` arrow, width 1
- `edge.attended`: light gray dotted, width 1
- `edge.reinforces`: green `#2D6B4A` solid, width 2, no arrow
- `edge.references`: yellow `#C8A43C` dotted, width 1

**Edge labels on hover** (per user decision -- clean by default):
- Default edge style: `'label': ''` (no label)
- On `mouseover` edge: set `label` to `edge.data('type')`, show with dark background
- On `mouseout` edge: clear label
- Guard against conflict with existing node hover: check a `nodeFocused` state variable before showing edge labels

**Layer toggle UI** (add to header area):
- Three toggle buttons: "Structure" (active), "Content" (active), "Intelligence" (active)
- CSS class `layer-btn` with De Stijl styling (dark bg, cream text, border on active)
- Click handler: `toggleLayer(layerName, isActive)` -- uses `cy.nodes('[layer="X"]').show()/.hide()` and connected edges
- Toggling a layer off hides both nodes AND their connected edges (unless the other end is visible)

**Preset view buttons** (add below layer toggles):
- Four buttons: "Room" (default), "Meetings", "Team", "Intel"
- CSS class `preset-btn` with De Stijl styling
- Presets are saved combinations of layer toggles + node filters:
  - Room Overview: Structure + Content visible, Intelligence hidden
  - Meeting Map: Structure visible, only meeting + speaker nodes in Content, artifacts hidden
  - Team Network: Only speaker + meeting nodes visible, SPOKE_IN/ATTENDED edges
  - Intelligence Map: All layers visible, content nodes dimmed (opacity 0.3), intelligence nodes full opacity
- Each preset button activates its combo, deactivates other presets

**Position persistence** (per user decision):
- On `dragfree` event: save node position to `localStorage` key `'graphPositions'`
- On graph load: read saved positions, apply via preset layout for nodes that exist in current graph. New nodes (not in saved positions) get auto-placed via CoSE on just those nodes.
- Clear stale IDs (saved positions for nodes not in current graph) on each load.

**Update detail panel** for new node types:
- Meeting node click: show meeting date, speakers list, decisions count, action items count
- Speaker node click: show role, role_type, connected meetings list
- Concept node click: show resolved/unresolved status, list of referencing artifacts

<task type="auto">
  <name>Task 2: Add timeline mode with chronological meeting layout and edge animations</name>
  <files>dashboard/index.html</files>
  <action>
Add timeline mode to the dashboard as a layout toggle (NOT a separate view -- per user decision).

**Timeline toggle button:**
- Add a "Timeline" button next to existing layout controls (or near the preset view buttons)
- Toggle between freeform (CoSE, current default) and timeline (preset) layout
- Active state styling consistent with other De Stijl buttons

**Timeline layout implementation:**
- Use Cytoscape.js `preset` layout with a `positions` function:
  - Meeting nodes: X = date mapped to pixel range (earliest date = left, latest = right), Y = top row (100px)
  - Section-group nodes: become horizontal bands -- fixed Y per section (evenly spaced below meetings row), X = center of timeline
  - Artifact nodes: X = their `created` date (or parent meeting date if filed from meeting), Y = their section's Y position
  - Speaker nodes: hidden in timeline mode (reduce clutter) -- use `cy.nodes('.speaker').hide()` when entering timeline
  - Concept nodes: hidden in timeline mode (too noisy)
- `animate: true`, `animationDuration: 300` for smooth transition
- On exiting timeline mode (switching back to CoSE): restore speaker/concept visibility based on current layer toggle state, re-run CoSE layout

**Date-to-X mapping function:**
- `dateToX(dateStr)`: parse YYYY-MM-DD, map to pixel range (100 to containerWidth - 100)
- Handle edge case: if only one meeting, center it
- Handle missing dates: position at left edge

**Section-to-Y mapping function:**
- `sectionToY(sectionId)`: map 8 sections to evenly spaced Y positions (200 to containerHeight - 100)
- Meetings row at Y=100 (top)

**Edge animations in timeline mode** (per user decision):
- REINFORCES edges: CSS animation class that pulses green (use Cytoscape.js `cy.style().update()` to add/remove animation)
- CONTRADICTS edges: CSS animation class that pulses red
- Implement via `setInterval` toggling edge opacity/width, or use Cytoscape.js `edge.animate()` for width oscillation
- Only animate when in timeline mode. Stop animation when switching back to freeform.

**Non-meeting node handling** (Claude's discretion per CONTEXT.md):
- Methodology artifacts without a meeting_date: use their `created` frontmatter date for X positioning. If no date, position at left edge.
- Section-group nodes: render as horizontal band labels on the Y axis (not as positioned boxes in the timeline space)
