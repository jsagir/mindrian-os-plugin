Create `scripts/generate-presentation.cjs` modeled on generate-export.cjs but producing 6 files instead of 1.

**Generator structure:**
1. Parse args: `node scripts/generate-presentation.cjs ROOM_DIR [--theme dark|light] [--output DIR]`
2. Default output: `ROOM_DIR/exports/presentation/`
3. Reuse the ROOM_DATA collection logic from generate-export.cjs (parseFrontmatter, SECTION_COLORS, section discovery, intelligence, graph, KuzuDB query, stats)
4. ADDITIONALLY collect:
   - MINTO.md content (for deck slides) -- read `ROOM_DIR/MINTO.md` if exists, parse into sections
   - Opportunity bank entries -- read `ROOM_DIR/opportunity-bank/*.md` if exists
   - Assets manifest -- read `ROOM_DIR/ASSET_MANIFEST.md` if exists
   - Team members -- read `ROOM_DIR/team/members/*/PROFILE.md` if exists
   - Video embed URL -- check STATE.md frontmatter for `video_url` field
   - Meeting count, speaker count (already in generate-export.cjs)
5. Build extended ROOM_DATA: `{ roomName, stage, generatedDate, theme, sections[], intelligence, graph, stats, kuzu, minto, opportunities[], assets[], team[], videoUrl }`
6. Read canvas-graph.js and graph-detail-panel.js via `fs.readFileSync` for inline embedding into graph.html and dashboard.html
7. For each of 6 templates in `templates/presentation/`:
   - Read template file
   - Replace `/*ROOM_DATA_PLACEHOLDER*/{}` with actual JSON
   - Replace `/*CANVAS_GRAPH_JS*/` with canvas-graph.js content (for graph.html, dashboard.html)
   - Replace `/*GRAPH_DETAIL_PANEL_JS*/` with graph-detail-panel.js content
   - Replace `{{ROOM_NAME}}` with ventureName
   - Replace `{{THEME}}` with 'dark' or 'light'
   - Replace `{{GENERATED_DATE}}` with ISO date
   - Write to output directory
8. Copy any room assets (images from ASSET_MANIFEST) to output/assets/ if they exist
9. Print summary: "Presentation generated: 6 views in {output}"

**Key decisions:**
- Zero npm dependencies (continues Phase 10 pattern)
- Templates are fully self-contained HTML (all CSS inline, JS inline, no external deps except Google Fonts CDN and marked.min.js CDN)
- canvas-graph.js is read from `lib/graph/canvas-graph.js` and injected as inline `<script>` (no external file reference -- self-contained)
- If a template file is missing, skip it with a warning (graceful degradation while templates are being built in Plans 02/03)

Insert after the HSI computation block (line ~117) and before the classify-insight exec (line ~120):

```bash
# -- Regenerate presentation views (background, non-blocking) --
# Only fires when room has exports/presentation/ directory (user has generated at least once)
if [ -n "$room_dir" ] && [ -d "${room_dir}/exports/presentation" ]; then
  node "${PLUGIN_ROOT}/scripts/generate-presentation.cjs" "$room_dir" --output "${room_dir}/exports/presentation" 2>/dev/null &
fi
```

**Key design decisions:**
- Only fires when `exports/presentation/` already exists (user must run generate-presentation manually first to opt in)
- Runs in background (&) so it does not block the hook
- No timeout needed (generate-presentation is pure file I/O, ~100ms)
- Respects the existing hook budget (3s total) -- this adds negligible overhead since it is background
- The git-ops step (if enabled) will pick up the regenerated HTML files on the NEXT commit cycle

This wires FILE-01's deferred "generate-presentation step" into the cascade.
