**Create `templates/presentation/insights.html`** ported from Align insights.html (355 lines) but templatized:

Sections generated from ROOM_DATA:
1. **Stats Overview**: animated counters for artifacts, meetings, speakers, sections, graph edges. Each counter animates from 0 to final value over 1.5s using requestAnimationFrame. IntersectionObserver triggers animation when scrolled into view.
2. **Section Health**: color-coded cards per section showing artifact count, empty/populated status, section color bar. Sorted by artifact count descending.
3. **Timeline**: if ROOM_DATA has meetings or artifacts with dates, render a vertical timeline with date markers and event descriptions. Falls back to section creation order if no dates.
4. **Intelligence Quadrant**: 2x2 matrix plotting sections by (artifact density) x (convergence score). Sections in each quadrant listed with colored dots.
   - Top-right: "Strong & Connected" (high artifacts + high convergence)
   - Top-left: "Rich & Isolated" (high artifacts + low convergence)
   - Bottom-right: "Sparse & Connected" (low artifacts + high convergence)
   - Bottom-left: "Needs Attention" (low artifacts + low convergence)
5. **Relationship Funnel**: from total artifacts -> connected artifacts -> converging themes -> contradictions. Funnel visualization using progressively narrower colored bars.
6. **Comparison Table**: sections as rows, columns for artifact count, edge count, convergence themes, gap severity. Sortable by column click.

Layout: single scrollable page, max-width 1200px centered, section cards with Bebas Neue titles.
Topbar: room name, "Back to Dashboard" link, stage tag.
Branding (PRES-09): meta tags, system instruction, header logo, footer with Mondrian color bar.
Theme: De Stijl dark default, `[data-theme="light"]` override block, `{{THEME}}` placeholder.

**Create `templates/presentation/diagrams.html`** ported from Align diagrams.html (1,375 lines) but templatized:

**SVG generation approach** (no external Graphviz binary -- pure JavaScript SVG):
Generate SVG diagrams from ROOM_DATA.graph using JavaScript:
- Calculate node positions using a simple force-directed layout (lighter than canvas-graph, just for static SVG)
- Or use a radial/hierarchical layout for cleaner static diagrams
- Draw SVG `<circle>` for nodes, `<line>` or `<path>` for edges, `<text>` for labels
- Edge styles match canvas-graph: INFORMS (gray), CONTRADICTS (dashed red), CONVERGES (dotted gold), ENABLES (blue arrow)

**Diagram types generated:**
1. **Ecosystem Map**: all nodes and edges, radial layout with sections as clusters
2. **Section Relationships**: one diagram per section showing that section's nodes and their cross-section connections
3. **Relationship Type Views**: one diagram per edge type (INFORMS only, CONTRADICTS only, etc.) -- shows only edges of that type
4. **Stakeholder Map** (if team data exists): speaker nodes connected to sections they consulted on

Each diagram rendered as a card with title, description, and SVG content.

**Theme toggle** (unique to diagrams view):
- Button in topbar toggles `data-theme` attribute between dark/light
- SVG elements use CSS variables for colors
- `[data-theme="light"] .diagram-svg svg` applies filter for better contrast
- Toggle state saved to localStorage

Topbar: room name, "Back to Dashboard" link, theme toggle button.
