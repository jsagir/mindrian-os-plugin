Create three command files. All commands should use Larry's voice (warm, concise, action-oriented).

**commands/help.md** — `/mindrian-os:help`:
YAML frontmatter: `name: help`, `description: See what Larry can help with -- commands tailored to where you are`, `allowed-tools: [Read, Glob]`.

Body instructions:
1. Read room/STATE.md to determine venture stage (Pre-Opportunity, Discovery, Validation, Design, Investment). If no room exists, stage = Pre-Opportunity.
2. Read references/methodology/index.md for full command list.
3. **Default behavior** (no flags): Larry recommends 2-3 commands most relevant to current state. E.g., if problem-definition is empty: "You haven't defined your problem yet. Start with `/mindrian-os:new-project` or tell me what you're working on." If problem exists but no market analysis: "Good problem definition. Now let's see who has this problem -- try `/mindrian-os:analyze-needs` (coming in next update)."
4. **With `--all` flag**: Show full command list grouped by venture stage. Mark commands not yet available with "(coming soon)". Use the methodology index for descriptions.
5. Always end with: "Or just talk to me. I don't need a command to help you think."

Progressive disclosure logic (from CONTEXT.md):
- Pre-Opportunity: show new-project, help, status, room (4 commands)
- Discovery: + explore-domains, analyze-needs, explore-trends (show as "coming soon")
- Validation: + challenge-assumptions, validate, think-hats
- Design: + structure-argument, scenario-plan, build-thesis
- Investment: + grade, assess-readiness, build-thesis

**commands/status.md** — `/mindrian-os:status`:
YAML frontmatter: `name: status`, `description: See where your project stands -- room state, venture stage, and what to do next`, `allowed-tools: [Read, Glob, Bash]`.

Body instructions:
1. If no room/ directory: "No project yet. Run `/mindrian-os:new-project` to get started."
2. If room/ exists: Run compute-state script (or read existing STATE.md) to get current state.
3. Display:
   - **Venture Stage**: Inferred stage with brief explanation. Larry confirms: "Looks like you're in Discovery -- you have a problem but haven't validated the market yet."
   - **Room Overview**: Table of sections with entry counts, status (Active/Empty/Well-developed), last updated
   - **Gaps**: Sections with no entries, framed as opportunities: "Your competitive-analysis room is empty. That's where `/mindrian-os:challenge-assumptions` would help."
   - **Suggested Next Action**: One specific recommendation based on gaps and stage
   - **Brain Status**: "Brain: not connected (using local references)" -- per CONTEXT.md, silent fallback with status indicator
4. Do NOT show tier information (build full-power version first per CONTEXT.md).

**commands/room.md** — `/mindrian-os:room`:
YAML frontmatter: `name: room`, `description: Manage your Data Room -- view sections, add rooms, export for investors`, `allowed-tools: [Read, Write, Bash, Glob]`.

Body instructions with subcommands:
1. **`/mindrian-os:room view`** (default, no subcommand): Display room overview with section-by-section detail. For each section: name, purpose (from ROOM.md), entry count, most recent entry title, starter questions if empty.
2. **`/mindrian-os:room add {name}`**: Create a new room or sub-room. Create directory + ROOM.md with purpose and starter questions. Larry confirms: "Added `room/team-execution/advisors/` -- good move, separate your advisory board from core team."
3. **`/mindrian-os:room export`**: Create a clean copy of the room tree in `export/` directory. Strip internal metadata (ROOM.md frontmatter, provenance lines). Keep clean markdown content only. Structure mirrors room/ but without STATE.md, USER.md. Result is investor-ready data room folder.
