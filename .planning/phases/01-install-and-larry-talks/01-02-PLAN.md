**hooks/run-hook.cmd** (~50 lines):
Polyglot bash/cmd wrapper. On Windows: detects Git Bash and delegates. On Unix: runs bash directly. Takes script name as argument, finds it in scripts/ directory relative to plugin root. Sets `CLAUDE_PLUGIN_ROOT` env var. Passes through all arguments. Must be executable (`chmod +x`).

**scripts/compute-state** (shared function, ~80 lines):
Bash script that scans a room/ directory and outputs STATE.md content. Logic:
1. Find all section directories under room/ (skip .git, hidden dirs)
2. For each section: count .md files (excluding ROOM.md), get last modified date, read first 5 lines of ROOM.md for section identity
3. Compute completeness (has entries = partially complete, has 3+ entries = well-developed, 0 entries = empty with starter questions available)
4. Detect cross-references: scan .md files for `[[section-name]]` or `room/section-name` patterns
5. Read USER.md if it exists
6. Infer venture stage from which sections have content (only problem-definition = Pre-Opportunity, problem + market = Discovery, etc.)
7. Output structured markdown STATE.md content (not JSON -- this is a markdown file)

STATE.md format:
```markdown
---
computed: {ISO timestamp}
venture_stage: {inferred stage}
total_entries: {N}
---
# Data Room State

## Sections
| Section | Entries | Status | Last Updated |
|---------|---------|--------|-------------|
| problem-definition | 3 | Active | 2026-03-20 |
...

## Gaps
- competitive-analysis: No entries. Consider /mindrian-os:challenge-assumptions
...

## Cross-References
- team-execution/advisors.md references legal-ip
...

## Suggested Next Action
{Based on venture stage and gaps}
```
Must be executable (`chmod +x`).

**scripts/session-start** (~60 lines):
Bash script called by hook on SessionStart. Logic:
1. Determine WORK_DIR (current directory where user is)
2. Check if room/ directory exists
3. If room/ exists: call compute-state to build context, read USER.md, read existing STATE.md
4. Build context string with room overview, user context, suggested actions
5. If room/ doesn't exist: context = "No room initialized. Suggest /mindrian-os:new-project to get started."
6. Escape context for JSON using `escape_for_json()` function (handle newlines, quotes, backslashes)
7. Output JSON: check for `CLAUDE_PLUGIN_ROOT` env var. If set: `{"hookSpecificOutput": {"hookEventName": "SessionStart", "additionalContext": "..."}}`. If not (Cursor): `{"additional_context": "..."}`.
Must be executable (`chmod +x`). Must complete in under 2 seconds.

**scripts/on-stop** (~40 lines):
Bash script called by hook on Stop. Logic:
1. Determine WORK_DIR, check if room/ exists
2. If room/ exists: call compute-state, write output to room/STATE.md
3. Output minimal JSON (no additionalContext needed for Stop hook, just success signal)
Must be executable (`chmod +x`).

**hooks/hooks.json**:
Configure two hooks following the pattern from 01-RESEARCH.md Example 3:
- SessionStart: matcher "startup|clear|compact", command calls run-hook.cmd with session-start, async: false, statusMessage: "Loading room context..."
- Stop: command calls run-hook.cmd with on-stop, timeout: 3000

IMPORTANT: All scripts use `set -euo pipefail`. All scripts use relative paths via `SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"` to find plugin root. No absolute paths in any script.

Create the `/mindrian-os:new-project` command file. This is the "onboarding" experience -- Larry does a deep exploration (5-10 min conversation) about the venture, then creates a tailored Data Room.

**commands/new-project.md**:
YAML frontmatter:
```yaml
name: new-project
description: Start a new venture project with Larry -- he'll explore your idea and create your Data Room
allowed-tools: [Read, Write, Bash, Glob]
```

Body (instructions for Claude when user invokes this command):

1. **Check for existing room**: If `room/` directory exists, warn user: "You already have a project in this workspace. MindrianOS supports one project per workspace in this version."

2. **Read USER.md** if it exists (returning user).

3. **Deep exploration phase** (5-10 minutes, Larry's voice):
   - Start with: "I'm Larry. What are you working on?"
   - Explore through questions (NOT a form -- a conversation):
     - What problem are they solving? Who has it?
     - What's their solution idea? How is it different?
     - Who are the customers? How big is the market?
     - What's the business model? How will they make money?
     - Who is on the team? What's their unfair advantage?
     - Any competitive landscape awareness?
     - Any legal/IP considerations?
   - Larry should push back on vague answers: "That's too broad. Pick the ONE customer who would pay for this tomorrow."
   - After sufficient exploration (user has shared enough for room context), Larry summarizes what he heard and proposes the room structure.

4. **Create room structure**:
   - Create `room/` directory
   - Create 8 base section directories per DD alignment (from 01-RESEARCH.md):
     - `room/problem-definition/`
     - `room/market-analysis/`
     - `room/solution-design/`
     - `room/business-model/`
     - `room/competitive-analysis/`
     - `room/team-execution/`
     - `room/legal-ip/`
     - `room/financial-model/`
   - Each section gets a `ROOM.md` with YAML frontmatter (`section`, `purpose`, `stage_relevance`, `default_methodologies`) and body with section description + 2-3 starter questions. Use the ROOM.md example from 01-RESEARCH.md as template. Tailor starter questions based on what the user shared (not generic).

5. **Create USER.md** at `room/USER.md`:
   - Capture: name (if shared), background, venture context, learning style observations, what they care about most, their language style.

6. **Create initial entries**: For sections where the user shared substantive content during exploration, create a brief entry file (e.g., `room/problem-definition/initial-exploration.md`) capturing the key points discussed. Include provenance metadata: `source: new-project exploration, date: {today}`.

7. **Compute initial STATE.md**: Run `scripts/compute-state` to generate the initial STATE.md. STATE.md is always computed from filesystem truth via the script, never generated directly by Claude.

8. **Create 00_Context/ for Cowork** (SURF-02): If the environment suggests Cowork (check for relevant env vars), also create `00_Context/` directory with a brief project summary.

9. **Close with next action**: Larry suggests what to work on first based on the conversation. E.g., "Your problem definition is solid but your competitive landscape is empty. Want to explore that, or should we dig deeper into your business model?"
