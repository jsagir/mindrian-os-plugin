**Create `scripts/file-asset` bash script** (ASSET-01, ASSET-02):

A bash script that files a binary asset into a room with a markdown wrapper. Usage:
```
scripts/file-asset <room_path> <file_path> <section> [--meeting <meeting_id>]
```

The script must:
1. **Validate inputs**: room_path exists and has STATE.md, file_path exists, section is one of the 8 known sections OR "meetings"
2. **Classify binary type** from extension:
   - pdf: .pdf
   - image: .png, .jpg, .jpeg, .gif, .svg, .webp
   - video: .mp4, .mov, .avi, .mkv, .webm
   - audio: .mp3, .wav, .m4a, .ogg, .flac, .aac
   - document: .pptx, .xlsx, .docx, .csv
   - archive: .zip, .tar, .gz
   - unknown: anything else
3. **Create assets directory**: `mkdir -p "$room_path/assets/$section/"`
4. **Copy binary file**: `cp "$file_path" "$room_path/assets/$section/"`
5. **Create markdown wrapper** at `$room_path/$section/{filename-without-ext}.md` with frontmatter:
   ```yaml
   ---
   type: {pdf|image|video|audio|document|archive}
   asset_path: assets/{section}/{original_filename}
   original_path: {absolute path of source file}
   file_size: {human-readable, e.g. "2.4 MB"}
   section: {section}
   created: {YYYY-MM-DD}
   meeting_id: {meeting_id if --meeting flag}
   ---
   # {filename}

   Binary asset filed to {section}.
   ```
6. **Update ASSET_MANIFEST.md**: Call the manifest update logic (see below)
7. **Git commit**: `bash "$SCRIPT_DIR/git-ops" commit "$room_path" "$room_path/assets/$section/$filename" "asset: $section/$filename"`
   Also commit the markdown wrapper: `bash "$SCRIPT_DIR/git-ops" commit "$room_path" "$wrapper_path" "asset-wrapper: $section/$filename"`
8. **Output**: Print `FILED:$section:$type:$filename` on success

For the manifest update (step 6), append or regenerate the ASSET_MANIFEST.md at room root. Use a simple approach: scan `$room_path/assets/` recursively for all files, collect metadata (type from extension, size via `stat`, section from parent dir, date from file mtime), and write:

```markdown
# Asset Manifest

> Auto-generated. Do not edit manually.

| File | Type | Section | Size | Date |
|------|------|---------|------|------|
| [filename](assets/section/filename) | pdf | market-analysis | 2.4 MB | 2026-03-30 |
...

Total: {N} assets ({total_size})
```

For `--meeting` flag: When the meeting_id is provided, the wrapper gets the meeting_id in frontmatter, and the asset_path uses the meeting directory instead: `meetings/{meeting_id}/{filename}`. No copy needed since file-meeting already copies audio to the meeting dir. The wrapper goes to `meetings/{meeting_id}/{filename-without-ext}-asset.md` and the manifest still gets an entry.

**Create `lib/core/asset-ops.cjs`** (ASSET-03):

Node.js wrapper following the room-ops.cjs pattern (execSync, zero npm deps). Exports:
- `fileAsset(roomDir, filePath, section, options)` - calls `scripts/file-asset` via execSync with 10s timeout. options: { meetingId }
- `readManifest(roomDir)` - reads and parses ASSET_MANIFEST.md, returns array of `{ file, type, section, size, date, path }` objects. Returns empty array if no manifest exists.
- `updateManifest(roomDir)` - calls `scripts/file-asset` with a `--manifest-only` flag (add this flag to the bash script) that just regenerates the manifest without filing a new asset. Useful for dashboard refresh.
- `getAssetsBySection(roomDir, section)` - convenience: filters readManifest() by section
- `getAssetsByType(roomDir, type)` - convenience: filters readManifest() by type (image, pdf, video, etc.)

The readManifest parser should parse the markdown table format: split by newlines, skip header rows (first 4 lines), parse pipe-delimited columns, extract link target from `[text](path)` pattern.
