  <name>Task 1: Create scripts/compute-team for TEAM-STATE.md knowledge landscape</name>
  <files>scripts/compute-team</files>
  <action>
Create `scripts/compute-team` as a bash script. Make it executable (chmod +x).

**Usage:** `scripts/compute-team <room_dir>`
**Output:** Writes `room/team/TEAM-STATE.md` directly (NOT stdout like compute-state).
**Exit:** 0 on success, 1 on error. If no team/ directory exists, exit 0 silently (no team yet).

**Script structure:**

1. **Parse args and validate:**
   ```bash
   ROOM_DIR="${1:-.}"
   TEAM_DIR="$ROOM_DIR/team"
   if [[ ! -d "$TEAM_DIR" ]]; then exit 0; fi
   ```

2. **Scan all PROFILE.md files for team roster:**
   - Find all `*/PROFILE.md` under team/ (glob: `$TEAM_DIR/*/*/PROFILE.md`)
   - For each, extract: name, roles (fall back to role), primary_role (fall back to role), status, affiliation, first_meeting, last_active, meetings_attended
   - Use grep/sed for field extraction (no YAML parser needed)
   - Handle both `roles: [mentor]` (Phase 7) and `role: mentor` (Phase 6 fallback)
   - Store in parallel arrays for tabulation

3. **Scan room/ sections for contribution counts:**
   - For each section dir (skip team/, meetings/, hidden dirs)
   - For each .md file in section (skip ROOM.md)
   - Extract speaker from frontmatter: try `attribution:` block first (grep for `^  speaker:` indented), fall back to flat `^speaker:` field
   - Build a contribution count: speaker-name -> section -> count
   - Track which sections each person has contributed to

4. **Compute expertise distribution:**
   - For each person, list sections they've contributed to with counts
   - Identify primary expertise (section with most contributions)

5. **Compute knowledge gaps:**
   - For each of the 8 topic sections, count unique contributors
   - Flag sections with 0 contributors as CRITICAL gap
   - Flag sections with 1 contributor as CONCENTRATION risk
   - Express gaps as missing perspectives, not missing work

6. **Compute missing perspectives:**
   - Check which roles from the 12-type taxonomy are NOT represented in team/
   - Specifically flag if no customer voice, no domain-expert, no legal perspective

7. **Compute role distribution:**
   - Count people per role (using primary_role)
   - Table: Role | Count | Members

8. **Compute activity patterns:**
   - For each person: status, meetings_attended, last_active
   - Calculate recency: if last_active is more than 30 days ago AND status is active, flag as "declining"
   - If last_active within 14 days, mark trend as "consistent"
   - Between 14-30 days: "occasional"

9. **Write TEAM-STATE.md:**
   Output to `$TEAM_DIR/TEAM-STATE.md` with this structure:

   ```markdown
   ---
   computed: {ISO timestamp}
   team_size: {N}
   active_members: {N with status=active}
   roles_represented: [{unique roles list}]
   ---
   # Team Knowledge Landscape

   ## Expertise Distribution

   | Person | Primary Expertise | Sections Contributed | Last Active |
   |--------|------------------|---------------------|-------------|

   ## Knowledge Gaps

   | Section | Contributors | Gap Assessment |
   |---------|-------------|----------------|

   ## Missing Perspectives

   - {bullet list of missing role perspectives}

   ## Role Distribution

   | Role | Count | Members |
   |------|-------|---------|

   ## Activity Patterns

   | Person | Status | Meetings | Last Active | Trend |
   |--------|--------|----------|-------------|-------|
   ```

   Keep the tone neutral and factual. This is context for Larry, not a report for users. No judgment language ("productive", "lazy"). Use knowledge-framing ("CRITICAL: No legal perspective" not "Nobody is doing legal work").

10. **Update PROFILE.md Contributions section** for each person:
    While scanning, also rebuild the `## Contributions` section in each PROFILE.md. Find the `## Contributions` marker (or `## Key Contributions` for Phase 6 profiles) and replace everything from that marker to the next `##` or EOF with a computed contributions table:

    ```markdown
    ## Contributions

    | Date | Section | Type | Artifact |
    |------|---------|------|----------|
    | 2026-03-15 | market-analysis | insight | [[market-analysis/2026-03-15-enterprise-growth.md]] |
    ```

    Extract date, section, and type from each artifact's frontmatter. Sort by date descending.

**Important constraints:**
- NEVER read STATE.md (circular dependency -- see Pitfall 2 from RESEARCH.md)
- Handle both Phase 6 flat frontmatter and Phase 7 attribution block
- No external dependencies -- bash builtins + grep/sed/find/date only
- TEAM-STATE.md is a CONTEXT TOOL. Never use words like "productive", "attendance rate", "performance"
  </action>

**compute-state changes:**

1. After the existing team count section (around line 107-109 where `team_count` is computed), add a call to compute-team:

   ```bash
   # Compute team intelligence (writes TEAM-STATE.md)
   SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
   if [ -d "$ROOM_DIR/team" ] && [ "$team_count" -gt 0 ]; then
     bash "$SCRIPT_DIR/compute-team" "$ROOM_DIR" 2>/dev/null || true
   fi
   ```

   Use `SCRIPT_DIR` to locate compute-team relative to compute-state (they are sibling scripts). The `|| true` ensures compute-state does not fail if compute-team has an issue.

2. Expand the Team section output (currently just "Team profiles: N") to include a brief summary from TEAM-STATE.md if it exists:

   ```bash
   if [ "$team_count" -gt 0 ]; then
     echo "## Team"
     echo ""
     echo "- **Team profiles:** $team_count"
     # Read roles from TEAM-STATE.md if available
     if [ -f "$ROOM_DIR/team/TEAM-STATE.md" ]; then
       roles_line=$(grep '^roles_represented:' "$ROOM_DIR/team/TEAM-STATE.md" | sed 's/^roles_represented: *//')
       active_count=$(grep '^active_members:' "$ROOM_DIR/team/TEAM-STATE.md" | sed 's/^active_members: *//')
       [ -n "$roles_line" ] && echo "- **Roles:** $roles_line"
       [ -n "$active_count" ] && echo "- **Active:** $active_count"
       # Show critical gaps if any
       gap_critical=$(grep 'CRITICAL' "$ROOM_DIR/team/TEAM-STATE.md" 2>/dev/null | head -3)
       if [ -n "$gap_critical" ]; then
         echo "- **Knowledge gaps detected** -- see team/TEAM-STATE.md"
       fi
     fi
     echo ""
   fi
   ```

3. Keep ALL existing compute-state logic unchanged. This is purely additive.
