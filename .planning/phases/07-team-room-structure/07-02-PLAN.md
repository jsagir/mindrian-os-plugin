  <name>Task 1: Expand file-meeting Step 5 to create full meeting archive package</name>
  <files>commands/file-meeting.md</files>
  <action>
In `commands/file-meeting.md`, rewrite Step 5 "Create Meeting Summary + Archive" to produce the full meeting package. Keep the existing transcript.md, summary.md, filed-to/, and compact root reference exactly as they are. ADD the following new files:

**1. speakers.md** -- Create after transcript.md:

```markdown
---
meeting_id: {YYYY-MM-DD-meeting-slug}
meeting_date: {YYYY-MM-DD}
---
# Speakers: {meeting_name}

| Speaker | Role | Segments | Profile |
|---------|------|----------|---------|
| {name} | {role} | {count} | [[team/{role-plural}/{slug}/PROFILE.md]] |
```

One row per confirmed speaker from Step 2. Profile links use the slug from create-speaker-profile.

**2. decisions.md** -- Create after summary.md:

```markdown
---
meeting_id: {YYYY-MM-DD-meeting-slug}
---
# Decisions: {meeting_name}

1. **{Decision summary}** -- {speaker} ({role})
   Filed to: [[{section}/{artifact-filename}.md]]
   Impact: {cascade_sections from the decision artifact}
```

Extract from the segments classified as `decision` in Step 3. If no decisions were made, write: "No explicit decisions were recorded in this meeting."

**3. action-items.md** -- Create after decisions.md:

```markdown
---
meeting_id: {YYYY-MM-DD-meeting-slug}
---
# Action Items: {meeting_name}

| Owner | Task | Deadline | Status |
|-------|------|----------|--------|
| {name} | {task} | {date or "not specified"} | open |
```

Extract from segments classified as `action-item`. Deadlines ONLY if explicitly stated in transcript -- never invent deadlines. All items start as `status: open`. If no action items, write: "No action items were identified in this meeting."

**4. metadata.yaml** -- Create as the LAST file in the archive (after all other data is known):

```yaml
meeting_id: {YYYY-MM-DD-meeting-slug}
meeting_name: {human-readable meeting name}
meeting_date: {YYYY-MM-DD}
source: {transcript | velma}
speakers:
  - name: {full name}
    role: {role}
    slug: {speaker-slug matching profile directory}
  - name: {full name}
    role: {role}
    slug: {speaker-slug}
topics:
  - {dominant topic 1}
  - {dominant topic 2}
decisions_count: {N}
insights_count: {N}
action_items_count: {N}
sections_touched:
  - {section-name}
  - {section-name}
has_audio: {true | false}
```

Topics are inferred from the dominant themes of filed segments. Speaker slugs MUST match the directory names created by create-speaker-profile (canonical slug source).

**5. Audio copy** -- If input was `--audio <path>`, copy the audio file into the meeting archive:
```
cp {audio-path} room/meetings/{YYYY-MM-DD-meeting-slug}/{original-filename}
```

**6. Meeting name inference + confirmation** -- Add to the beginning of Step 5 (or end of Step 1 metadata inference if not already there):
- Larry proposes a meeting name based on the dominant topic/purpose from classified segments
- User confirms or changes the name
- The confirmed name is slugified for the meeting_id: lowercase, hyphens, no special chars, max 50 chars
- This meeting_id is used in all archive paths AND in all attribution blocks from Step 4

**7. Update the archive directory listing** at the top of Step 5 to show the complete structure:
```
room/meetings/YYYY-MM-DD-{meeting-name}/
  transcript.md
  summary.md
  speakers.md          # NEW
  decisions.md         # NEW
  action-items.md      # NEW
  metadata.yaml        # NEW
  {audio-filename}     # NEW (only if --audio)
  filed-to/
```

**8. Past meeting lookup** -- Add a new substep at the end of Step 5 or as a note in Setup:
"When Larry needs to reference past meetings (e.g., 'Lawrence mentioned this 3 meetings ago'), grep metadata.yaml files across room/meetings/:
- By speaker: `grep -rl '{speaker-slug}' room/meetings/*/metadata.yaml`
- By topic: `grep -rl '{topic}' room/meetings/*/metadata.yaml`
- By date range: scan meeting_date fields in metadata.yaml files
This provides fast targeted lookups without indexing."
