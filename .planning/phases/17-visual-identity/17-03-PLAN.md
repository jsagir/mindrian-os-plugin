**Add to visual-ops.cjs:**

**`generateMermaidRoom(sections)`** — Takes array of `{ name, entryCount, stage, edges: [{ to, type }] }`. Returns valid Mermaid `graph TD` syntax with:
- Node labels: `PD[Problem Definition 80%]`
- Edge labels: `-->|INFORMS|`
- Style directives using De Stijl hex colors: `style PD fill:#A63D2F,color:#F5F0E8`
- Empty sections styled differently (dashed border or lighter fill)

**`generateMermaidGraph(nodes, edges)`** — Takes KuzuDB/LazyGraph data (arrays of `{id, type, label}` nodes and `{from, to, type}` edges). Returns Mermaid `graph LR` with typed edges colored by relationship type.

**`generateMermaidChain(steps)`** — Takes methodology chain steps `[{name, framework, status}]`. Returns Mermaid `graph TD` showing the chain flow: diagnose -> framework -> apply -> file -> cross-ref -> graph-update.

**`wrapMermaidHtml(mermaidSyntax, title)`** — Returns complete HTML document string with:
- Mermaid.js CDN script
- De Stijl dark theme (background #0D0D0D, text #F5F0E8)
- Title in header
- The Mermaid diagram in a `<pre class="mermaid">` block
- Mermaid initialized with `{ theme: 'dark', themeVariables: { primaryColor: '#1E3A6E', ... } }`

**`generateMermaidBlock(mermaidSyntax)`** — Returns markdown fenced code block: ````mermaid\n${syntax}\n```` for embedding in .md files.

**Create `scripts/render-viz`:**

Bash script (executable, with shebang) that:
1. Accepts sub-command: `render-viz room [roomDir]`, `render-viz graph [roomDir]`, `render-viz chain [roomDir]`
2. Calls Node.js to generate Mermaid syntax using visual-ops functions + room-ops/state-ops for data
3. Wraps in HTML using wrapMermaidHtml
4. Writes to `${TMPDIR:-/tmp}/mindrian-viz/diagram.html`
5. Opens in browser using serve-dashboard pattern (xdg-open / wslview / open)
6. For `render-viz room`: reads room STATE.md and section dirs to build section data, then calls generateMermaidRoom
7. For `render-viz graph`: reads .lazygraph data via lazygraph-ops query, then calls generateMermaidGraph
8. For `render-viz chain`: reads latest .reasoning/ run artifact, then calls generateMermaidChain
9. Prints Mermaid syntax to stdout as fallback (so it can be captured in markdown)

Make the script executable: `chmod +x scripts/render-viz`

```markdown
Open rich visual diagrams in the browser. Falls back to Mermaid code blocks in text output.

## Sub-commands

- `/mos:visualize room` — Room structure flowchart (sections, stages, cross-references)
- `/mos:visualize graph` — Knowledge graph view (LazyGraph nodes and edges)
- `/mos:visualize chain` — Latest methodology chain flowchart
- `/mos:visualize timeline` — Meeting timeline (Gantt-style)

## Usage

Larry runs `scripts/render-viz <subcommand>` to generate and open the diagram.
If browser is unavailable, outputs Mermaid code block that renders in GitHub/Obsidian.

User: "Show me the room structure as a diagram"
Larry: Runs `/mos:visualize room` — opens browser with interactive Mermaid flowchart

User: "What does my knowledge graph look like?"
Larry: Runs `/mos:visualize graph` — opens browser with force-directed graph view
```
