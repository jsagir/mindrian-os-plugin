Create `scripts/generate-export.cjs` -- a Node.js script (CommonJS, zero npm deps, pure built-ins only per Phase 10 decision) that reads a room/ directory and generates a self-contained HTML export.

**Script structure:**

```javascript
#!/usr/bin/env node
'use strict';
const fs = require('fs');
const path = require('path');
const { execSync } = require('child_process');
```

**Arguments:** `node scripts/generate-export.cjs [ROOM_DIR] [OUTPUT_PATH]`
- ROOM_DIR defaults to `./room`
- OUTPUT_PATH defaults to `room/exports/YYYY-MM-DD-{room-name}.html`

**Main flow:**

1. **Validate room exists.** Exit with error if no room/ directory.

2. **Read STATE.md** from room directory. Parse YAML frontmatter to extract:
   - `venture_name` (fallback: directory name)
   - `stage` (fallback: "Discovery")
   Use the regex-based frontmatter parser pattern from existing codebase (no yaml library).

3. **Discover sections.** Scan room/ for subdirectories. Skip hidden dirs, `meetings/`, `team/`, `exports/`, `STATE.md`, `MINTO.md`, `ROOM-INTELLIGENCE.md`. For each section directory:
   - Count .md files (excluding ROOM.md, STATE.md, MINTO.md)
   - Read first artifact's title from frontmatter (for summary)
   - If no artifacts, read ROOM.md purpose field for summary
   - Map section name to color using SECTION_COLORS constant (same as build-graph). Unknown sections get `#D4CFC7` (gray).
   - Determine lightText: true for sections with dark bg colors (problem-definition, solution-design, business-model, personas)

4. **Read all markdown content.** For each section, for each .md artifact (excluding ROOM.md, STATE.md, MINTO.md):
   - Read full file content
   - Store as `{ sectionId, content }` for embedding as `<script type="text/markdown">` blocks
   - If multiple artifacts in a section, concatenate with `\n\n---\n\n` separator, using a single content-{sectionId} script tag

5. **Run analyze-room** for intelligence data. Execute: `bash scripts/analyze-room "${roomDir}"` with 5s timeout. Parse output lines:
   - Lines starting with `GAP:` -> `{ section, message, severity }`
   - Lines starting with `CONVERGENCE:` -> split themes by comma
   - Lines starting with `CONTRADICTION:` -> `{ sections, message, severity }`
   - If analyze-room fails or times out, use empty intelligence: `{ gaps: [], convergence: [], contradictions: [] }`

6. **Run build-graph** for graph data. Execute: `bash scripts/build-graph "${roomDir}" "${tempGraphPath}"` with 10s timeout. Read resulting JSON. If fails, use empty graph: `{ elements: { nodes: [], edges: [] } }`.

6b. **Query LazyGraph (KuzuDB) for richer relationship data.** If `room/.lazygraph/` exists:
   - Import `{ openGraph, closeGraph, queryGraph, graphStats }` from `lib/core/lazygraph-ops.cjs`
   - Open the graph: `const { db, conn } = await openGraph(roomDir)`
   - Query all edges: `MATCH (a)-[r]->(b) RETURN type(r) AS relType, a.id AS src, b.id AS tgt, a.title AS srcTitle, b.title AS tgtTitle`
   - Query stats: `await graphStats(conn)`
   - Close: `await closeGraph(db)`
   - Merge KuzuDB edges into graph.elements.edges (LazyGraph has typed relationships: INFORMS, CONTRADICTS, CONVERGES, ENABLES, INVALIDATES, BELONGS_TO, REASONING_INFORMS)
   - Add `kuzu` field to ROOM_DATA: `{ available: true, artifacts: N, sections: N, edges: { INFORMS: N, CONTRADICTS: N, ... } }`
   - If `.lazygraph/` doesn't exist or kuzu import fails, set `kuzu: { available: false }` and continue (graceful degradation)
   - Use try/catch with `closeGraph` in finally block (open-use-close pattern per Phase 15 decision)

7. **Compute stats:**
   - artifacts: total .md files across all sections (excluding metadata files)
   - meetings: count directories in room/meetings/
   - speakers: count directories in room/team/members/ (or 0)
   - sections: number of discovered sections
   - empty: sections with 0 artifacts

8. **Build ROOM_DATA object** matching the contract from Plan 01 interfaces.

9. **Read the template** from `dashboard/export-template.html` (path relative to script location).

10. **Inject data:**
    - Replace `/*ROOM_DATA_PLACEHOLDER*/{"roomName":"Sample Room",...}` with `/*ROOM_DATA_PLACEHOLDER*/` + JSON.stringify(ROOM_DATA)
    - Replace `<!-- CONTENT_SCRIPTS_PLACEHOLDER -->` with concatenated `<script id="content-{sectionId}" type="text/markdown">{content}</script>` blocks
    - Replace the topbar title span content with room name
    - Replace the footer date with actual generation date

11. **Write output.** Ensure `room/exports/` directory exists. Write the HTML file. Print success message with file path.

**Error handling:** Try/catch around each data collection step. Always produce output even with partial data (graceful degradation). Log warnings to stderr for failed steps.

**Section grid class assignment:** For the `gridClass` field, assign CSS grid positions based on section index:
- Index 0: `cell-0` (grid-column: 1/2, grid-row: 1/3) -- large left cell
- Index 1: `cell-1` (grid-column: 2/4, grid-row: 1/2) -- wide right top
- Stage badge: always at (grid-column: 3/4, grid-row: 2/3)
- Index 2: `cell-2` (grid-column: 2/3, grid-row: 2/3)
- Index 3+: fill remaining grid positions
- The template JS handles the actual grid layout; gridClass is a hint.

**IMPORTANT:** Zero npm dependencies. Use only `fs`, `path`, `child_process` built-ins.
