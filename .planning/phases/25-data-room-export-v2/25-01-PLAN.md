**CRITICAL: Read BOTH reference files fully before writing. The PolygonEye reference IS the design spec.**

**Structure:**

1. **Head section:**
   - Google Fonts: DM Sans (labels/nav), Playfair Display (headlines), Source Serif 4 (body), JetBrains Mono (data/code)
   - CDN scripts: marked.js (markdown), cytoscape.js (graph), mermaid.js (diagrams)
   - MindrianOS meta tags (generator, creator, author, og:, system instruction comment) -- copy exact pattern from current `dashboard/index.html` lines 6-20
   - All CSS inline (no external stylesheets)

2. **CSS -- De Stijl Design System (copy from PolygonEye reference, adapt for dynamic sections):**
   - Root variables: `--mondrian-red: #C23B22`, `--mondrian-blue: #1B3B6F`, `--mondrian-yellow: #E8B931`, `--mondrian-black: #1A1A1A`, `--mondrian-white: #F7F3ED`, `--mondrian-gray: #D4CFC7`, `--accent-green: #2D6B4A`, `--accent-sienna: #B5602A`
   - Topbar: 56px height, sticky, black bg, DM Sans 13px uppercase title, yellow stage badge, view switcher buttons (Overview/Intelligence/Graph)
   - Mondrian grid: `grid-template-columns: 2fr 1fr 1fr`, `grid-template-rows: 260px 200px 180px 160px`, gap:0, 5px solid black borders, cell assembly animation (0.6s fade-in with staggered delays)
   - Cell typography: section label (DM Sans 10px uppercase), title (Playfair 20px), summary (Source Serif 13px), meta (JetBrains 10px)
   - Document view: 3-column layout (200px rail + flex content + 220px TOC). Rail has vertical label, back button. Content has frontmatter infobox, markdown styles. TOC sticky.
   - **Document sidebar navigation** (from Black Domain reference): When in document view, add a navigation panel between the rail and content area. 240px wide. Shows all sections as nav items with color dots, entry counts. Clicking a section loads that section's documents without returning to overview. Active section highlighted with left border in section color.
   - Intelligence view: white bg, section groups (GAPS with red badges, CONVERGENCE with yellow badges, CONTRADICTIONS with red dashed border cards). Intel cards with section label + message + confidence badge. Validation table if data exists.
   - Graph view: flex layout, Cytoscape container + 260px sidebar with legend (section squares, meeting diamonds, speaker circles, edge types) + stats.
   - Footer: black bg, JetBrains 11px, "Generated by MindrianOS . PWS Methodology . {date}"
   - Print styles: hide topbar/footer/nav, show documents only
   - Responsive: tablet (2-col grid, hide TOC), mobile (1-col, hide sidebars)
   - Zero border-radius everywhere (De Stijl rule)

3. **HTML body:**
   - Topbar with room name + stage badge + view switcher
   - `#view-overview`: Mondrian grid. **DYNAMIC:** Instead of hardcoded cells, the JS `buildGrid()` function reads `window.ROOM_DATA.sections` and generates cells. Include a `<div class="mondrian-grid" id="mondrian-grid"></div>` container. The grid layout adapts to section count: 8-9 sections use the full 3x4 grid from PolygonEye. Fewer sections collapse rows. More sections add rows.
   - `#view-document`: Doc view with rail + sidebar nav + content + TOC (structure from PolygonEye lines 1032-1047, PLUS the sidebar nav panel)
   - `#view-intelligence`: Intel view built dynamically from `window.ROOM_DATA.intelligence`
   - `#view-graph`: Graph container + sidebar (structure from PolygonEye lines 1252-1302)
   - Footer with generation date
   - **Content injection point:** A comment `<!-- CONTENT_SCRIPTS_PLACEHOLDER -->` where the generate-export script will insert `<script type="text/markdown">` blocks

4. **JavaScript (inline, at bottom of body):**
   - `window.ROOM_DATA` placeholder: Insert `var ROOM_DATA = /*ROOM_DATA_PLACEHOLDER*/{"roomName":"Sample Room","stage":"Discovery","generatedDate":"2026-01-01","sections":[],"intelligence":{"gaps":[],"convergence":[],"contradictions":[]},"graph":{"elements":{"nodes":[],"edges":[]}},"stats":{"artifacts":0,"meetings":0,"speakers":0,"sections":0,"empty":0}};`
   - Section color map: `const SECTION_COLORS = { 'problem-definition': '#C23B22', 'solution-design': '#1B3B6F', 'market-analysis': '#F7F3ED', 'business-model': '#2D6B4A', 'competitive-analysis': '#F7F3ED', 'team-execution': '#F7F3ED', 'meeting': '#D4CFC7', 'personas': '#C23B22', 'legal-ip': '#F7F3ED', 'financial-model': '#F7F3ED' };`
   - Light text sections (dark bg cells): `const DARK_BG_SECTIONS = ['problem-definition', 'solution-design', 'business-model', 'personas'];`
   - `buildGrid()`: Reads ROOM_DATA.sections, creates Mondrian cells dynamically. Each cell has: section label, title (from summary), entry count badge, onclick to openDoc. The stage badge cell is always position [row2, col3] with yellow bg. Assign grid-column/grid-row CSS based on section index using the PolygonEye asymmetric layout. Stagger animation-delay by 50ms per cell.
   - `switchView(view)`: Toggle active class on view containers and topbar buttons. Init graph on first graph view open.
   - `buildSidebarNav()`: Populate the document sidebar with all sections from ROOM_DATA. Each item shows color dot + section name + entry count.
   - `openDoc(section)`: Read `<script id="content-{section}">` tag, strip frontmatter, render with `marked.parse()`, populate TOC from h2/h3, set rail color, update sidebar active state.
   - `buildIntelligence()`: Read ROOM_DATA.intelligence, build gaps cards (red badges), convergence tags (yellow badges), contradiction cards (red dashed border). If no intelligence data, show "No intelligence data available."
   - `initGraph()`: Create Cytoscape instance with ROOM_DATA.graph elements. This must be VISUALLY APPEALING, IMMERSIVE, and PLAYFUL — not a dry technical graph. Design principles:
     **Node styling (beautiful, not clinical):**
     - Section nodes: 90x90 rectangles, 4px black Mondrian border, filled with section color, uppercase DM Sans label. On hover: subtle glow (box-shadow with section color at 40% opacity), scale 1.05.
     - Artifact nodes: 50x50 rectangles, lighter shade of parent section color, Source Serif title. On hover: expand to show full title + first line of content.
     - Meeting nodes: 60x60 diamonds, gold (#D4A843), black border. On hover: show date + participant count.
     - Speaker nodes: 50x50 ellipses, deep blue (#1E3A6E), white border. On hover: show name + role.
     - Concept/keyword nodes: 30x30 circles, yellow (#C8A43C), glow animation (subtle pulse). CLICKABLE: clicking a keyword highlights all connected nodes (the "keyword filter" pattern from milken-twin appendix links).
     **Edge styling (semantic and beautiful):**
     - CONVERGES: solid gold (#C8A43C), width 2.5, subtle glow on hover, arrow
     - CONTRADICTS: dashed red (#C23B22), width 2.5, red glow on hover, NO arrow (bidirectional)
     - INFORMS: solid gray (#5C5A56), width 1.5, thin arrow
     - REINFORCES: solid green (#2D6B4A), width 2, arrow
     - SPOKE_IN/FILED_TO: dotted gray, width 1, very subtle
     **Interactive behaviors (immersive):**
     - Hover node: highlight ALL connected edges + nodes, dim everything else to 0.15 opacity. Transitions 0.3s.
     - Click node: open detail panel (right sidebar slides in) with full content, cross-references as clickable links.
     - Click keyword/concept node: filter mode — only show nodes connected to that keyword. Everything else fades. Click background to reset.
     - Click edge: show edge label tooltip with relationship type and connected node names.
     - Double-click section node: zoom to that section's subgraph (all artifacts + edges within).
     **Layout:**
     - < 13 nodes: grid layout (predictable, clean)
     - 13+ nodes: CoSE force-directed with nodeRepulsion 15000, idealEdgeLength 140, gravity 0.25
     - Animate layout on load (1.2s, easing)
     **Graph header overlay:** Room name + node/edge count + "Powered by MindrianOS" in small JetBrains Mono text at bottom-right of graph view.
   - `initMermaid()`: After markdown rendering, find code blocks with `mermaid` class, render with `mermaid.run()`.
   - DOMContentLoaded: call `buildGrid()`, `buildIntelligence()`.

**Key differences from PolygonEye (which was hardcoded):**
- Grid cells are generated from ROOM_DATA, not hardcoded HTML
- Intelligence view is generated from ROOM_DATA.intelligence, not hardcoded
- Graph data comes from ROOM_DATA.graph, not a const
- Section count is variable (the template handles 4-12+ sections)
- Document sidebar nav is new (from Black Domain reference)

Similarly, `<!-- CONTENT_SCRIPTS_PLACEHOLDER -->` gets replaced with actual markdown script tags.
