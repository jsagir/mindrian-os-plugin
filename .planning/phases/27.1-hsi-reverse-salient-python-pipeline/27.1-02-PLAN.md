**Create `scripts/check-hsi-deps`** -- a bash script that checks HSI dependency availability.

```bash
#!/usr/bin/env bash
# check-hsi-deps -- Report HSI tier availability
# Exit codes: 0=tier1+, 1=no python, 2=no sklearn
# Output to stdout: "tier:0", "tier:1", or "tier:2"

set -euo pipefail

# Check Python 3
if ! command -v python3 &>/dev/null; then
  echo "tier:0"
  exit 1
fi

# Check sklearn (Tier 1 minimum)
if ! python3 -c "import sklearn" 2>/dev/null; then
  echo "tier:0"
  exit 2
fi

# Check sentence-transformers (Tier 1 full)
has_st=false
if python3 -c "import sentence_transformers" 2>/dev/null; then
  has_st=true
fi

# Check Pinecone (Tier 2)
has_pinecone=false
if python3 -c "import pinecone" 2>/dev/null && [ -n "${PINECONE_API_KEY:-}" ]; then
  has_pinecone=true
fi

if $has_pinecone && $has_st; then
  echo "tier:2"
  exit 0
elif $has_st; then
  echo "tier:1"
  exit 0
else
  # sklearn present but no embeddings -- can still do LSA-only
  echo "tier:1"
  exit 0
fi
```

Make it executable: `chmod +x scripts/check-hsi-deps`

**Extend `scripts/post-write`** -- add Step 11 (HSI computation) AFTER Step 10 (git-ops).

Insert the following block after the Step 10 git-ops section (after line 137 in current file). This is Step 11 in the cascade:

```bash
# -- Step 11: HSI computation + KuzuDB bridge (background, non-blocking) --
# Fires AFTER git-ops. Only runs when:
#   a) File is a .md file in a room
#   b) room_dir is set and has .lazygraph
#   c) Python + sklearn are available (check-hsi-deps exits 0)
#   d) At least 2 artifacts exist in the room
if [[ "$FILE_PATH" == *.md ]] && [ -n "$room_dir" ] && [ -d "${room_dir}/.lazygraph" ]; then
  (
    # Check HSI deps are available (silent check)
    if bash "${SCRIPT_DIR}/check-hsi-deps" >/dev/null 2>&1; then
      # Run HSI computation
      python3 "${PLUGIN_ROOT}/scripts/compute-hsi.py" "$room_dir" 2>/dev/null || true

      # Run reverse salient detection
      python3 "${PLUGIN_ROOT}/scripts/detect-reverse-salients.py" "$room_dir" 2>/dev/null || true

      # Write results to KuzuDB
      if [ -f "${room_dir}/.hsi-results.json" ]; then
        node "${PLUGIN_ROOT}/scripts/hsi-to-kuzu.cjs" "$room_dir" 2>/dev/null || true
      fi
    fi
  ) &
fi
```

Key design decisions (per HSI-04):
- The ENTIRE HSI block runs in a subshell `( ... ) &` so it's fully non-blocking
- Each Python script has `|| true` so failures don't propagate
- The hsi-to-kuzu.cjs only runs if .hsi-results.json actually exists
- No timeout -- HSI can take 10-30s for large rooms, that's fine in background
- check-hsi-deps must exit 0 for any of the Python to fire (graceful skip)
- Placed AFTER git-ops (Step 10) so the committed artifact is the one analyzed
