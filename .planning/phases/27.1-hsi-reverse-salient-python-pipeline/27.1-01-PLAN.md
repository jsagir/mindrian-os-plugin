**Create `scripts/compute-hsi.py`** -- ported from V4 `hsi_semantic_surprise.py` (537 lines) and V2 `compute_lsa.py` (259 lines), adapted for room/ markdown reading.

The script MUST:
1. Accept `room_dir` as first argument (e.g., `python3 scripts/compute-hsi.py /path/to/room`)
2. Accept optional `--tier` argument (1 or 2, default 1). Tier 0 is handled by existing bash `analyze-room` -- this script only handles Tier 1 and Tier 2. (Per HSI-05)
3. Accept optional `--threshold` argument (default 0.30)
4. Accept optional `--output` argument (default `{room_dir}/.hsi-results.json`)

**Room artifact reading:**
- Walk all subdirectories of room_dir looking for .md files
- Skip STATE.md, ROOM.md, MINTO.md, and any files in .lazygraph/ or .git/
- For each .md file: extract frontmatter (use regex, NOT PyYAML -- match plugin pattern), extract body text after `---` block
- Build artifact list: `[{id: "section/filename-without-ext", section: "section-name", title: "from # heading", path: "relative", text: "body content"}]`
- Require minimum 2 artifacts to proceed (exit 0 with empty results otherwise)

**Cache system:**
- Read `{room_dir}/.hsi-cache.json` if it exists
- Store MD5 hash of each artifact's content
- If ALL hashes match cached hashes, skip computation and exit 0 (no changes)
- After computation, write updated hashes to cache

**LSA structural similarity** (port from V4 `compute_lsa_similarity` and V2 `compute_lsa.py`):
- `TfidfVectorizer(stop_words='english', max_features=500)` (V4 uses 500, lighter than V2's 2000 -- room artifacts are smaller than papers)
- `TruncatedSVD(n_components=min(80, n-1, n_features))`
- `cosine_similarity` on reduced matrix
- Clamp to [0, 1]

**Semantic similarity** (per HSI-05 3-tier):
- Tier 1: `from sentence_transformers import SentenceTransformer; model = SentenceTransformer('all-MiniLM-L6-v2')` -- 80MB, CPU-only
  - `model.encode(texts)` then `cosine_similarity`
- Tier 2: Read Pinecone config from env vars `PINECONE_API_KEY` and `PINECONE_INDEX`
  - Use `pinecone` SDK to query existing embeddings for artifact IDs
  - Fall back to Tier 1 if Pinecone not configured or query fails

**OM-HMM integrative thinking score** (port from V4 `compute_omhmm_score`):
- Copy the keyword lists `_INTEGRATIVE_KEYWORDS` and `_FEATURE_PATTERNS` from V4
- Score each artifact: `0.5 * likelihood_ratio * 100 + 0.3 * (feature_diversity / 8) * 100 + 0.2 * min(complexity_ratio * 50, 100)`

**HSI matrix computation** (port from V4 `execute` method):
- For each pair (i, j): `semantic_surprise = abs(semantic_sim - lsa_sim)`
- `integrative_factor = sqrt(omhmm_i * omhmm_j) / 100`
- `innovation_differential = 0.6 * semantic_surprise + 0.4 * integrative_factor`
- Classify: `structural_transfer` if lsa > semantic, else `semantic_implementation`
- `breakthrough_potential = 0.7 * differential + 0.3 * min(lsa, semantic)` (from V2)

**Output:**
- Write `.hsi-results.json` to room_dir following the schema in interfaces block above
- Include top 20 pairs sorted by hsi_score descending
- Print summary to stderr (pair count, top score, tier used)

**Error handling:**
- If sklearn not installed: print "HSI requires scikit-learn. Run: pip install -r requirements-hsi.txt" to stderr, exit 1
- If sentence-transformers not installed (Tier 1): print warning, try Tier 2, else exit 1
- All imports guarded with try/except

**Create `scripts/detect-reverse-salients.py`** -- ported from V2 `detect_reverse_salients.py` (287 lines), adapted for room sections.

The script MUST:
1. Accept `room_dir` as first argument
2. Read `{room_dir}/.hsi-results.json` (output of compute-hsi.py)
3. Accept optional `--threshold` (default 0.30) and `--top-n` (default 20)

**Cross-section detection logic** (V2 port):
- Group artifacts by section
- For each cross-section pair (section_A artifacts vs section_B artifacts):
  - Find the highest HSI score between any artifact in A and any artifact in B
  - If above threshold AND both individual sims > 0.20 (min_similarity): this is a reverse salient
- Classify each: `structural_transfer` or `semantic_implementation` (from V2 `classify_opportunity`)
- Score breakthrough potential: `0.7 * novelty + 0.3 * feasibility` (from V2 `score_breakthrough_potential`)
- Generate innovation thesis string (from V2 template)

**Output:**
- Update `.hsi-results.json` by adding/replacing the `reverse_salients` array
- Print summary to stderr

**Create `scripts/hsi-to-kuzu.cjs`:**

A Node.js CJS script that reads `.hsi-results.json` and writes edges to KuzuDB.

Usage: `node scripts/hsi-to-kuzu.cjs /path/to/room`

The script MUST:
1. Accept room_dir as first argument
2. Read `{room_dir}/.hsi-results.json` -- exit 0 silently if file doesn't exist
3. Import `openGraph`, `closeGraph`, `queryGraph` from `../lib/core/lazygraph-ops.cjs`
4. Use the open-use-close pattern (try/finally) as established in Phase 15

**HSI_CONNECTION edge creation:**
- For each pair in `hsi_pairs` where `hsi_score > 0.3`:
  - MERGE `HSI_CONNECTION` edge between the two artifact IDs
  - Set all properties: hsi_score, lsa_sim, semantic_sim, surprise_type, breakthrough_potential, tier
  - Use the `esc()` helper for string values (import from lazygraph-ops or redefine locally)

**REVERSE_SALIENT edge creation:**
- For each entry in `reverse_salients`:
  - MERGE `REVERSE_SALIENT` edge between source_section and target_section Section nodes
  - Set properties: differential_score, innovation_type, source_artifact, target_artifact, innovation_thesis
  - Ensure Section nodes exist first (MERGE Section node)

**Cleanup before write:**
- Before creating new edges, DELETE all existing HSI_CONNECTION and REVERSE_SALIENT edges (fresh recomputation each run)
- `MATCH (a)-[r:HSI_CONNECTION]->(b) DELETE r`
- `MATCH (s1)-[r:REVERSE_SALIENT]->(s2) DELETE r`

**Output:**
- Print to stderr: "HSI: wrote {N} connection edges, {M} reverse salient edges"
- Exit 0 on success

**Error handling:**
- If .hsi-results.json is malformed or empty, exit 0 silently
- If KuzuDB open fails, print error to stderr, exit 1
- Always close DB in finally block
