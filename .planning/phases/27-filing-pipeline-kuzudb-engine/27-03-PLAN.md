    **Extend indexArtifact in lazygraph-ops.cjs** -- after the existing assumption extraction block (added in Plan 01), add meeting-aware indexing:

    1. **Detect meeting-sourced artifacts**: Check if `methodology` equals `file-meeting` (already extracted as variable).

    2. **If file-meeting artifact**:
       a. Extract meeting_name, meeting_date, meeting_id from frontmatter:
          - `meeting_name = extractFrontmatter(content, 'meeting_name')`
          - `meeting_date = extractFrontmatter(content, 'meeting_date')`
          - For meeting_id: extract from attribution block. Parse `meeting_id:` line after `attribution:`. Use a regex: `content.match(/attribution:[\s\S]*?meeting_id:\s*["']?([^"'\n]+)/)`
          - If no meeting_id found, construct from meeting_date: `meeting_date + '-meeting'`
       b. Create/merge Meeting node: `MERGE (m:Meeting {id: '${meetingId}'}) ON CREATE SET m.name = ..., m.date = ...`
       c. Create SEGMENT_OF edge: extract `segment_type` from frontmatter.
          `MATCH (a:Artifact {id: '${artifactId}'}), (m:Meeting {id: '${meetingId}'}) MERGE (a)-[:SEGMENT_OF {segment_type: '${segmentType}', confidence: 0.8}]->(m)`

    3. **Speaker indexing from attribution block**:
       a. Extract speaker name: `content.match(/attribution:[\s\S]*?speaker:\s*["']?([^"'\n]+)/)`
       b. Extract speaker role: `content.match(/attribution:[\s\S]*?role:\s*["']?([^"'\n]+)/)`
       c. Extract profile_path: `content.match(/attribution:[\s\S]*?profile_path:\s*["']?([^"'\n]+)/)`
       d. If speaker name found:
          - Speaker ID = `speaker/` + speakerName.toLowerCase().trim().replace(/\s+/g, '-')
          - MERGE Speaker node with name, role, profile_path
          - MERGE SPOKE_IN edge from Speaker to Meeting

    4. **CONSULTED_ON edge creation**:
       - The artifact's `section` (parent directory) indicates which section the speaker consulted on
       - `MATCH (sp:Speaker {id: '${speakerId}'}), (sec:Section {name: '${section}'}) MERGE (sp)-[:CONSULTED_ON]->(sec)`
       - This creates the speaker-to-section expertise mapping automatically

    - [4] SPOKE_IN edge connects speaker to meeting
    - [5] CONSULTED_ON edge connects speaker to section
    - [6] Non-meeting artifact (methodology != file-meeting) does NOT create Meeting/Speaker nodes
    - [7] graph.json from build-graph-from-kuzu.cjs contains meeting and speaker nodes

    Use test-room-meeting fixtures from Plan 01. Follow test-phase-15.sh pattern.
