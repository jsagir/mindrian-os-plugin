  <action>
    Extend `lib/core/lazygraph-ops.cjs`:

    1. **Update EDGE_TYPES array** -- add 'SEGMENT_OF', 'SPOKE_IN', 'CONSULTED_ON', 'HAS_ASSUMPTION', 'ASSUMPTION_IMPACTS'

    2. **Extend initSchema(conn)**:
       - Add `CREATE NODE TABLE IF NOT EXISTS Meeting(id STRING PRIMARY KEY, name STRING, date STRING, source STRING, speakers_count INT64, decisions_count INT64, action_items_count INT64)`
       - Add `CREATE NODE TABLE IF NOT EXISTS Speaker(id STRING PRIMARY KEY, name STRING, role STRING, role_type STRING, profile_path STRING)`
       - Add `CREATE NODE TABLE IF NOT EXISTS Assumption(id STRING PRIMARY KEY, claim STRING, status STRING, source_artifact STRING, created STRING)`
       - Add edge tables: `SEGMENT_OF(FROM Artifact TO Meeting, segment_type STRING, confidence DOUBLE DEFAULT 0.5)`, `SPOKE_IN(FROM Speaker TO Meeting)`, `CONSULTED_ON(FROM Speaker TO Section, meeting_count INT64 DEFAULT 1)`, `HAS_ASSUMPTION(FROM Artifact TO Assumption)`, `ASSUMPTION_IMPACTS(FROM Assumption TO Section)`

    3. **Add migrateSchema(conn)** function -- called after initSchema in openGraph:
       - `ALTER TABLE Artifact ADD artifact_id STRING DEFAULT ''`
       - `ALTER TABLE Artifact ADD pipeline STRING DEFAULT ''`
       - `ALTER TABLE Artifact ADD pipeline_stage INT64 DEFAULT 0`
       - Each ALTER wrapped in try/catch (idempotent -- column may already exist)
       - NOTE: KuzuDB 0.11.x cannot ALTER REL tables to add properties. Confidence on existing edges (INFORMS, ENABLES, INVALIDATES) stays in app layer. New edge tables (SEGMENT_OF) include confidence natively.

    4. **Extend indexArtifact** -- after existing wikilink/enables/invalidates processing:
       - Parse `assumptions:` from frontmatter. The assumptions block is YAML array format. Use a simple regex parser: match lines starting with `  - claim:` to extract each assumption.
       - For each assumption: compute Assumption ID as `crypto.createHash('sha256').update(claim.toLowerCase().trim()).digest('hex').slice(0, 12)`
       - MERGE Assumption node: `{id, claim, status, source_artifact: artifactId, created}`
       - MERGE HAS_ASSUMPTION edge from Artifact to Assumption
       - For each `impacts` section: MERGE ASSUMPTION_IMPACTS edge from Assumption to Section

    5. **Add indexMeeting(conn, roomDir, meetingDir)** function:
       - Read `metadata.yaml` from meetingDir
       - Extract: meeting_name, meeting_date, source using extractFrontmatter-style parsing (YAML, not frontmatter -- read raw file, extract key: value lines)
       - Meeting ID = `meeting/` + basename of meetingDir
       - MERGE Meeting node
       - Return `{id, name, date}`

    6. **Add indexSpeaker(conn, meetingId, speaker)** function:
       - speaker = `{name, role, role_type, profile_path}` object
       - Speaker ID = `speaker/` + name.toLowerCase().replace(/\s+/g, '-')
       - MERGE Speaker node
       - MERGE SPOKE_IN edge from Speaker to Meeting
       - Return `{id, name}`

    7. **Update graphStats** -- add Meeting, Speaker, Assumption counts to nodes object. Add new edge types to the edge counting loop.

    8. **Update module.exports** -- add indexMeeting, indexSpeaker, migrateSchema

    9. **Update openGraph** -- call migrateSchema(conn) after initSchema(conn)

    CRITICAL: All changes use IF NOT EXISTS / MERGE patterns. Existing data is NEVER dropped. The esc() function is used for ALL string values in Cypher queries.
