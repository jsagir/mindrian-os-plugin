  <action>
    **lib/core/proactive-intelligence.cjs** -- Proactive intelligence persistence with repeat suppression:

    1. `persistIntelligence(roomDir, analyzeOutput)`:
       - `analyzeOutput` is the raw stdout string from `analyze-room`
       - Parse lines: GAP, CONVERGE, CONTRADICT patterns
       - Load existing `.proactive-intelligence.json` from roomDir (or empty `{insights: [], cross_room: []}`)
       - For each parsed insight:
         - Check if same insight already exists (match on type + section or type + term)
         - If exists: increment `times_shown`, update `last_seen`
         - If new: add with `first_seen: new Date().toISOString(), times_shown: 0`
       - Write back to `{roomDir}/.proactive-intelligence.json`
       - Return `{persisted: N, new: M, suppressed: S}`

    2. `loadIntelligence(roomDir)`:
       - Read `.proactive-intelligence.json` from roomDir
       - Return parsed JSON or `{insights: [], cross_room: []}` if file missing

    3. `shouldSuppress(insight)`:
       - Return true if `insight.times_shown >= 3` (user has seen it 3+ times)
       - This is used by session-start to filter what Larry surfaces

    4. `addCrossRoomRelationship(roomDir, sourceRoom, targetRoom, sharedConcepts)`:
       - Load existing intelligence
       - Check for duplicate (same source+target rooms)
       - If exists: update sharedConcepts (union), update detected timestamp
       - If new: add entry
       - Write back

    Module.exports: `{persistIntelligence, loadIntelligence, shouldSuppress, addCrossRoomRelationship}`

    **scripts/cross-room-detect.cjs** -- Cross-room concept detection:

    1. Takes `roomDir` as process.argv[2] (the room that just had a filing)
    2. Takes `workspaceDir` as process.argv[3] (defaults to process.cwd())
    3. Reads `.rooms/registry.json` from workspaceDir. If missing, exit 0.
    4. For the current room: extract all unique noun phrases / keywords from artifact titles in KuzuDB
       - Open KuzuDB: `MATCH (a:Artifact) RETURN a.title, a.section`
       - Extract keywords: split titles on spaces, filter words > 3 chars, lowercase, collect unique
    5. For each OTHER room in registry (skip current room):
       - Check if `{otherRoom}/.lazygraph` exists
       - If yes: open that room's KuzuDB, query artifact titles, extract keywords
       - Find intersection of keyword sets
       - If intersection has 3+ shared concepts: record as cross-room relationship
    6. For each detected cross-room relationship:
       - Call `proactiveIntelligence.addCrossRoomRelationship(roomDir, ...)`
    7. Close all KuzuDB connections (open-use-close per room)
    8. Output JSON summary to stdout: `{scanned: N, relationships: M, concepts: [...]}`

    CRITICAL: Each room's KuzuDB is opened and closed independently. Never hold two write connections. Use read-only queries (MATCH only, no MERGE). Wrap everything in try/catch -- if any room fails, skip it and continue.

    Performance: This is a BACKGROUND process. No timeout constraint, but keep it reasonable (<10s for 5 rooms).
  </action>
