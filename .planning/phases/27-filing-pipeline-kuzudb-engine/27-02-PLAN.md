  <action>
    **lib/core/artifact-id.cjs** -- Stable artifact ID computation and frontmatter injection:

    1. `computeArtifactId(roomDir, section, title, created)`:
       - `const roomName = path.basename(roomDir);`
       - `const input = roomName + ':' + section + ':' + title + ':' + created;`
       - Return `crypto.createHash('sha256').update(input).digest('hex').slice(0, 12)`

    2. `injectArtifactId(filePath, roomDir)`:
       - Read file content. If no frontmatter (no `---` block), return early.
       - If `artifact_id:` already present in frontmatter, return early (idempotent).
       - Extract section (parent dir basename), title (first # heading or filename), created (date field from frontmatter).
       - Compute artifact_id via computeArtifactId.
       - Insert `artifact_id: {hash}` line after the first `---` line in frontmatter.
       - Write file back. Return `{artifact_id, injected: true}`.

    3. `injectPipelineProvenance(filePath, pipeline, stage, requires, provides)`:
       - Read file. If no frontmatter, return early.
       - If `pipeline:` already present, return early (idempotent).
       - Insert pipeline provenance fields after artifact_id (or after first ---):
         ```
         pipeline: {pipeline}
         pipeline_stage: {stage}
         pipeline_requires: [{requires array as YAML}]
         pipeline_provides: [{provides array as YAML}]
         ```
       - Write file back. Return `{injected: true}`.

    All file writes use `fs.writeFileSync`. All frontmatter manipulation is string-based (split on `---`, manipulate lines, rejoin). Do NOT use gray-matter for writes -- it reformats content.

    **scripts/build-graph-from-kuzu.cjs** -- KuzuDB-sourced Cytoscape JSON builder:

    1. Takes roomDir as argv[2], outputPath as argv[3] (defaults to `{roomDir}/.presentation/graph.json`)
    2. Opens KuzuDB via lazygraph-ops.openGraph(roomDir)
    3. Queries all nodes:
       - `MATCH (a:Artifact) RETURN a.id, a.title, a.section, a.methodology, a.created`
       - `MATCH (m:Meeting) RETURN m.id, m.name, m.date`
       - `MATCH (s:Speaker) RETURN s.id, s.name, s.role`
       - `MATCH (s:Section) RETURN s.name, s.label`
       - `MATCH (a:Assumption) RETURN a.id, a.claim, a.status`
    4. Queries all edges: `MATCH (a)-[r]->(b) RETURN a.id AS src, type(r) AS relType, b.id AS tgt`
    5. Builds Cytoscape JSON format matching existing build-graph output structure:
       - `elements.nodes[]` with `{data: {id, label, group, type, ...}, classes: type}`
       - `elements.edges[]` with `{data: {id, source, target, type, label}, classes: type}`
       - Section nodes as group nodes (type: 'group', same colors as build-graph SECTION_COLORS)
       - Use the same 8 section colors from build-graph: problem-definition=#A63D2F, market-analysis=#C8A43C, etc.
    6. Closes KuzuDB, writes JSON to outputPath
    7. If .lazygraph/ does not exist, exit 0 silently (graceful degradation)
    8. Wrap main in async IIFE with try/catch. On error, write to stderr, exit 0 (never fail the hook chain).

    Use the open-use-close pattern (try/finally with closeGraph).
  </action>
