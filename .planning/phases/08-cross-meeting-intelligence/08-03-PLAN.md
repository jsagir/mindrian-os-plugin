YAML frontmatter:

```yaml
title: Cross-Meeting Intelligence Detection
description: Protocols for convergence detection, contradiction tracking, and action item lifecycle across meetings
tier: 0
used_by:
  - commands/file-meeting.md (Step 0, Step 4, Step 6)
```

Sections:

**1. Action Item Triage Protocol (Step 0)**
- Load `room/action-items.md` if it exists
- Present open items in a quick table: | # | Owner | Task | From Meeting |
- Framing (per user's specific idea): "3 open items from your last meeting. Quick check -- any done?"
- User responses: mark done (by number, e.g., "1,3") / skip / review all
- When marking done: update the SOURCE meeting's action-items.md (change `open` to `done`), not the aggregated file. The aggregated file gets rebuilt by compute-state.
- If no open items or no action-items.md: skip Step 0 silently.

**2. Cross-Reference During Filing Protocol (Step 4 Enhancement)**
- During segment filing (Step 4), for each segment being filed, compare against remaining open action items
- If a segment looks like progress on an open item, surface it: "This looks like progress on {Owner}'s '{Task}'. Mark as done?"
- Use Larry's judgment to match -- not exact text matching. An action item about "Review TAM analysis" could be addressed by a segment discussing "Updated market size projections."
- Only surface high-confidence matches (Larry should be at least 70% confident)

**3. Cross-Meeting Convergence Detection Protocol (Step 6 Enhancement)**
- After single-meeting cross-relationship scan, perform cross-meeting scan
- Scan protocol:
  1. Extract key topics/themes from the current meeting's filed segments
  2. Grep `topics:` fields in ALL `room/meetings/*/metadata.yaml` files
  3. For each topic appearing in 3+ meetings (including current), generate a convergence signal
  4. Present in Larry's voice: "Market validation has been raised in 4 of your last 6 meetings. This is becoming a central theme." (per user's framing)
- Convergence signals are written to the meeting summary AND tracked in room-level state

**4. Cross-Meeting Contradiction Detection Protocol (Step 6 Enhancement)**
- Load summaries from last 10 meetings (or all if fewer than 10) -- per Claude's discretion recommendation
- Compare current meeting claims against prior meeting claims using Larry's LLM reasoning
- Severity-based flagging (per locked decision):
  - HIGH-impact (financials, strategy, key decisions): Flag immediately during filing with Larry's voice: "Lawrence said TAM was $190M in the March meeting, but now says $120M. Which is current?"
  - LOW-impact (opinions, preferences): Collect in meeting summary's Contradictions section only
- When a contradiction is surfaced and user resolves it (confirms which claim is current), the resolution is noted

**5. Enhanced Meeting Summary Sections**
- Meeting summary gains two new sections (added to the summary-template structure):
  - `## Convergence Signals` -- topics recurring across 3+ meetings, with counts and meeting references
  - `## Cross-Meeting Contradictions` -- contradictions detected against prior meetings (not just within-meeting)
- These sections appear AFTER the existing sections (Key Decisions, Insights Filed, etc.)

**6. Context Window Management**
- Limit cross-meeting scanning to last 10 meetings max (prevents context overload -- per research pitfall #1)
- Use metadata.yaml grep to PRE-FILTER relevant meetings by topic/speaker BEFORE loading full summaries
- Only load full summary.md for meetings that share topics with current meeting
