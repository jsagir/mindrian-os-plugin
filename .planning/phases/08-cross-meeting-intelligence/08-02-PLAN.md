<task type="auto">
  <name>Task 1: Create compute-meetings-intelligence script</name>
  <files>scripts/compute-meetings-intelligence</files>
  <action>
Create a new bash script `scripts/compute-meetings-intelligence` that scans all meeting archives to produce two output files.

Script header and conventions:
- `#!/usr/bin/env bash`
- `set -e` (NO pipefail -- grep-heavy script, Phase 7 decision)
- Usage: `scripts/compute-meetings-intelligence <room_dir>`
- Exit silently if no meetings/ directory or zero meetings
- Use `SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"` pattern

**Section 1: Topic Frequency (Convergence Signals)**
- Scan all `metadata.yaml` files in `room/meetings/*/`
- Extract `topics:` arrays from each metadata.yaml
- Count topic frequency across meetings using associative arrays
- Track first-seen and latest-seen dates per topic
- Topics appearing in 3+ meetings become Active Convergence Signals (per locked decision)

**Section 2: Contradiction Aggregation**
- Scan all `summary.md` files in `room/meetings/*/`
- Look for `## Contradictions Detected` sections
- Extract contradiction entries (topic, claims, speakers, status)
- Aggregate into Active Contradictions table
- Track `unresolved` vs `resolved` status

**Section 3: Action Item Aggregation**
- Scan all `action-items.md` files in `room/meetings/*/`
- Parse markdown tables (skip header rows, extract Owner|Task|Deadline|Status columns)
- Aggregate all items with their source meeting_id (from directory name)
- Separate into open and done (recently completed)
- Write to `room/action-items.md` with computed timestamp frontmatter, total_open, total_done counts
- Format: table with | Owner | Task | Source Meeting | Date | Status |

**Section 4: Write MEETINGS-INTELLIGENCE.md**
- Write to `room/MEETINGS-INTELLIGENCE.md`
- YAML frontmatter: computed timestamp, meetings_analyzed count
- Sections (matching the structure from RESEARCH.md):
  1. **Active Convergence Signals** -- table: Theme | Meetings | First Seen | Latest
  2. **Active Contradictions** -- table: Topic | Claim A | Claim B | Speakers | Status
  3. **Meeting Timeline** -- table: Date | Name | Speakers | Decisions | Action Items (from metadata.yaml)
- Note: Team Patterns section (recurring concerns, influence, role gaps) is computed by compute-team extension, not this script. This script references TEAM-STATE.md for the full picture.

Use `|| true` after grep commands that may legitimately find no matches. Handle empty meetings gracefully (write "No convergence signals detected yet" etc.).

Make the script executable: `chmod +x scripts/compute-meetings-intelligence`.
