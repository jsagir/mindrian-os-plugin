  <files>lib/mcp/tool-router.cjs, bin/mindrian-mcp-server.cjs</files>
  <action>
Create `lib/mcp/tool-router.cjs` with the authoritative command-to-tool mapping from RESEARCH.md. Export `registerRouterTools(server, roomDir, pluginRoot)` and `ALL_TOOL_COMMANDS` (flat array of all 41 command strings for parity checking).

Register 6 router tools on the server:

1. **`data_room`** — commands: status, list-sections, analyze, compute-state, get-state, new-project, setup, update, help, suggest-next
   - inputSchema: `{ command: z.enum([...]), section: z.string().optional() }`
   - Dispatch: status/list-sections/analyze/compute-state/get-state call core modules (room-ops, state-ops). new-project/setup/update/help/suggest-next return the command's reference markdown for Claude to execute conversationally.
   - annotations: `{ readOnlyHint: false }` (mixed read/write)

2. **`methodology`** — commands: lean-canvas, think-hats, structure-argument, beautiful-question, build-knowledge, challenge-assumptions, validate, map-unknowns, diagnose, score-innovation, explore-domains, analyze-needs, user-needs
   - inputSchema: `{ command: z.enum([...]), context: z.string().optional().describe('Focus area or venture aspect') }`
   - Dispatch: Load `references/methodology/{command}.md`, combine with room state from stateOps.getState(roomDir), return as structured text for Claude to use as methodology guide
   - Include compact Larry personality in tool description

3. **`analysis`** — commands: analyze-systems, analyze-timing, find-bottlenecks, root-cause, systems-thinking, macro-trends, explore-trends, explore-futures, dominant-designs, scenario-plan
   - inputSchema: `{ command: z.enum([...]), context: z.string().optional() }`
   - Same dispatch pattern as methodology: load reference + room state

4. **`intelligence`** — commands: find-connections, build-thesis, compare-ventures, research, deep-grade, grade, leadership
   - inputSchema: `{ command: z.enum([...]), context: z.string().optional() }`
   - Same dispatch pattern. Grade/deep-grade also load assessment-philosophy.md from personality refs.

5. **`meeting`** — commands: file-meeting, pipeline
   - inputSchema: `{ command: z.enum([...]), transcript: z.string().optional(), date: z.string().optional() }`
   - file-meeting: loads meeting filing protocol reference + room state
   - pipeline: loads pipeline reference + room state

6. **`export`** — commands: export, radar
   - inputSchema: `{ command: z.enum([...]), format: z.string().optional() }`
   - Dispatch to respective reference + room state

For ALL router tools:
- Each tool description should be concise (under 200 chars) and mention Larry by name
- Use `safeReadFile()` from core index.cjs for all file reads
- Return `{ content: [{ type: 'text', text: ... }] }` per MCP tool response format
- If reference file not found, return helpful error with available commands
- If room state unavailable, note it but do not error (room may not exist yet)

NOTE: `suggest-next` is in commands/ but was missing from research mapping. Add it to `data_room` router group.

Wire into `bin/mindrian-mcp-server.cjs`:
- Add `require('../lib/mcp/tool-router.cjs')` import
- Call `registerRouterTools(server, roomDir, pluginRoot)` before transport.connect()
