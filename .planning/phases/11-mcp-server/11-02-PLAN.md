<task type="auto">
  <name>Task 1: Create MCP Resources for room:// URI scheme</name>
  <files>lib/mcp/resources.cjs, bin/mindrian-mcp-server.cjs</files>
  <action>
Create `lib/mcp/resources.cjs` exporting `registerResources(server, roomDir)`.

Register these resources:

1. **`room-state`** (static) — `room://state`
   - Returns contents of room/STATE.md via `stateOps.getState(roomDir)`
   - mimeType: `text/markdown`
   - If no STATE.md, return "No room initialized. Run /mindrian-os:new-project to create one."

2. **`room-sections`** (static) — `room://sections`
   - Returns JSON listing of all discovered sections with metadata (name, type, label, color)
   - Uses `discoverSections(roomDir)` from section-registry
   - mimeType: `application/json`

3. **`room-section`** (template) — `room://section/{sectionName}`
   - Uses `ResourceTemplate` with `list` callback that returns all discovered sections
   - Handler reads all .md files in `path.join(roomDir, sectionName)` and concatenates them
   - Validates sectionName exists as a directory, returns error text if not
   - mimeType: `text/markdown`

4. **`room-meetings`** (static) — `room://meetings`
   - Lists subdirectories of `room/meetings/` (each is a meeting date folder)
   - Returns JSON array of meeting folder names
   - If no meetings/ dir, return "No meetings filed yet."
   - mimeType: `application/json`

5. **`room-intelligence`** (static) — `room://intelligence`
   - Returns contents of `room/MEETINGS-INTELLIGENCE.md` if it exists
   - If missing, return "No meeting intelligence generated yet. File meetings first."
   - mimeType: `text/markdown`

Wire into `bin/mindrian-mcp-server.cjs`:
- Add `require('../lib/mcp/resources.cjs')` import
- Call `registerResources(server, roomDir)` after registerRouterTools
