Create `lib/chat/chat-context.js` -- a pure function module that builds Larry's system prompt from ROOM_DATA:

1. `buildRoomContext(roomData)` function:
   - Extract room name from `roomData.state.name` or `roomData.roomName`
   - Extract governing thought from `roomData.minto` if present
   - List room sections with artifact counts from `roomData.state.sections`
   - Summarize graph structure: node count, edge count, edge type distribution
   - If `roomData.currentView` is set, include view-specific context (e.g., "User is viewing the knowledge graph")
   - Return a string, max ~2000 chars (truncate long sections)

2. `buildSystemPrompt(roomContext)` function:
   - Embed Larry voice DNA excerpt (hardcode the essential personality rules inline - conversational, provocative, concise, 3-8 sentences, signature openers, reframe technique)
   - Wrap roomContext as "Room Context" section
   - Add instruction: "You are Larry, embedded in a MindrianOS Data Room. Answer questions about THIS room's content. You can call tools to highlight graph clusters and filter views."
   - Return full system prompt string

Export both functions. Use `var` and IIFE pattern matching canvas-graph.js style (no ES modules - these run in browser).

Create `lib/chat/chat-panel.js` -- embeddable vanilla JS chat panel:

1. `ChatPanel` constructor takes `(containerEl, options)`:
   - `options.roomData` -- the ROOM_DATA object
   - `options.onToolCall` -- callback when Larry invokes a tool (for Plan 32-02)
   - `options.model` -- defaults to 'claude-sonnet-4-20250514'

2. API key management:
   - On init, check `localStorage.getItem('mos-api-key')`
   - If no key: show input field with "Enter your Anthropic API key to chat with Larry" placeholder, a Save button, and a note "Key stored in your browser only - never sent to any server except Anthropic"
   - On save: `localStorage.setItem('mos-api-key', key)`, show chat interface
   - If key exists: show chat interface immediately with a small "key" icon that reveals key management (clear/change)

3. Chat UI (inject DOM into containerEl):
   - Collapsible panel: small "Chat with Larry" button (fixed bottom-right, De Stijl styled with Mondrian yellow accent)
   - When expanded: 360px wide, 500px tall panel with:
     - Header: "Larry" + minimize button
     - Message list: scrollable, auto-scroll on new messages
     - Messages styled: user (right-aligned, blue bg), Larry (left-aligned, dark bg)
     - Input: textarea + send button, Enter to send (Shift+Enter for newline)
   - Collapsed state persisted in `localStorage.getItem('mos-chat-open')`

4. Streaming Anthropic API calls:
   - POST to `https://api.anthropic.com/v1/messages`
   - Headers: `{ 'Content-Type': 'application/json', 'x-api-key': key, 'anthropic-version': '2023-06-01', 'anthropic-dangerous-direct-browser-access': 'true' }`
   - Body: `{ model, max_tokens: 1024, system: systemPrompt, messages: conversationHistory, stream: true }`
   - Parse SSE stream: read `event: content_block_delta` for text deltas, render incrementally
   - On `event: message_stop`, finalize message
   - On error (401): show "Invalid API key" and re-show key input
   - On error (other): show error message in chat

5. Message rendering:
   - Parse Larry's responses as simple markdown (bold, italic, code blocks, lists)
   - Use a minimal inline markdown renderer (regex-based, ~30 lines - no library)

6. Conversation history:
   - Maintain `messages` array in memory (not persisted)
   - Cap at 20 messages to stay within token limits
   - Include system prompt on every request (not in messages array)

Style the panel using CSS variables from the existing De Stijl theme (--bg-primary, --bg-secondary, --bg-card, --text-primary, --border-color, --mondrian-yellow). All CSS injected inline via JS (no external stylesheet needed).
