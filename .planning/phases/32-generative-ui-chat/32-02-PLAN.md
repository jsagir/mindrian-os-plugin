2. `executeToolCall(toolName, toolInput, context)` function:
   - `context` has `{ graph, roomData, chatPanel }` (graph = window.__mosGraph, etc.)
   - For `highlightCluster`: call `context.graph.highlightCluster(toolInput.group)`, return `{ success: true, message: 'Highlighted ' + toolInput.group }`. If no graph instance, return `{ success: false, message: 'Graph not available on this view' }`
   - For `filterEdgeType`: call `context.graph.highlightCluster(toolInput.edgeType)` (highlightCluster already handles edge types), return result object
   - For `showInsight`: return `{ success: true, rendered: true, component: { type: 'insight-card', ...toolInput } }` -- the chat panel will render this as an inline card

3. `renderComponent(component)` function -- the json-render pattern (GENUI-01):
   - Takes a component spec object and returns an HTML string
   - For `type: 'insight-card'`: render a styled card div with title, items as rows (label: value with optional color dot), and footer
   - For `type: 'list'`: render a styled list
   - For `type: 'table'`: render a styled table
   - Style using the same CSS variables (--bg-card, --border-color, --text-primary, etc.)
   - This is a minimal declarative UI renderer -- Larry generates JSON, this renders it

Export: `TOOL_DEFINITIONS`, `executeToolCall`, `renderComponent`.

   - Style with a subtle border-left accent (Mondrian yellow)

5. If `GenerativeTools` is not available (module not loaded), skip tool definitions in the API call -- chat still works as plain text.
