  <name>Task 1: Create lib/graph/canvas-graph.js -- Universal Canvas 2D Renderer</name>
  <files>lib/graph/canvas-graph.js</files>
  <action>
Port the Milken Twin Canvas graph renderer (lines 922-1254 of /home/jsagi/Mindrian/milken-twin/vercel-app/public/index.html) into a reusable, data-driven module. READ that source file first to get the exact implementation patterns.

**Create `lib/graph/canvas-graph.js` as a class-based vanilla JS module (no npm deps, embeddable via script tag).**

The class `CanvasGraph` accepts:
- `container` -- DOM element to mount into (creates its own canvas)
- `graphData` -- graph.json object (Cytoscape format from build-graph-from-kuzu.cjs)
- `options` -- optional overrides (onNodeClick callback, theme colors)

**Data adaptation layer (consume graph.json):**
- Parse `graphData.elements.nodes` and `graphData.elements.edges`
- For each node: extract `data.id`, `data.label`, `data.color`, `data.section`, `data.layer`, `classes` (section-group, artifact, meeting, speaker, assumption)
- Node radius: section-group nodes get r=24. Artifact/meeting/speaker/assumption nodes get r based on degree centrality: count edges touching that node, then `r = Math.max(8, Math.min(20, 6 + degree * 2))`
- For each edge: extract `data.source`, `data.target`, `data.type`
- Build a nodeMap (id -> node) and adjacency list for fast neighbor lookups

**Canvas setup (per GRAPH-01):**
- Create canvas element, append to container
- Retina scaling: `canvas.width = W * 2; canvas.height = H * 2; ctx.scale(2, 2)` -- exactly like Milken Twin
- ResizeObserver on container to handle resize

**Force simulation (per GRAPH-01):**
Port the exact Milken Twin simulation with these forces:
- Gravity to center: `vx += (cx - x) * 0.0008`
- Node-node repulsion: `force = 800 / (d * d)` for all pairs
- Edge attraction: `force = (d - 100) * 0.003` along each edge
- Velocity damping: `vx *= 0.85; vy *= 0.85`
- Boundary clamping: keep nodes within canvas bounds
- Initialize positions in circular spread with random jitter (same as source)

**Node rendering (per GRAPH-02):**
- Circular nodes (not rectangles -- this is the Canvas style, different from the Cytoscape export)
- Color from node.data.color at 60% opacity (append `99` hex to color string)
- Section-group nodes rendered larger (r=24) with thicker stroke
- Labels: show for large nodes (r > 12) or hovered/connected nodes. Font: '600 13px Inter,sans-serif' for large, '400 11px Inter,sans-serif' for small

**Animated particles (per GRAPH-03):**
- Spawn a new particle on a random edge every 8 frames
- Particle travels from source to target with `speed = 0.008 + Math.random() * 0.012`
- Draw particle as small circle (r=2.5) with alpha based on `Math.sin(t * Math.PI)`
- Draw trail particle 3% behind at half size, lower alpha
- Color from source node's color

**Hover interaction (per GRAPH-04):**
- Track mouse position via mousemove on canvas
- Find hovered node: distance from mouse to node center < node.r + 8
- When hovering: connected nodes (direct neighbors via any edge) stay at full opacity, everything else dims to alpha 0.15
- Edges connected to hovered node: strokeStyle brightened, lineWidth 2
- Non-connected edges: very faint (rgba with 0.06 alpha)
- Set cursor to 'pointer' when hovering a node

**Glow rings (per GRAPH-05):**
- On hovered node AND on section-group nodes (core nodes): draw radial gradient ring
- Glow radius: node.r + 10 + pulseR
- Gradient: from color at 40% alpha to transparent
- Ambient pulse on section-group nodes when nothing is hovered: `pulseR = Math.sin(frameCount * 0.03 + nodeIndex * 0.5) * 3`

**Edge type styling (per GRAPH-08):**
- INFORMS: thin (lineWidth 0.8), gray (#888), small arrowhead at target
- CONTRADICTS: dashed (setLineDash([6, 4])), red (#FF4B4B), lineWidth 1.5
- CONVERGES: dotted (setLineDash([2, 4])), gold (#C8A43C), lineWidth 1.2
- ENABLES: solid, blue (#4A90D9), lineWidth 1.5, arrowhead at target
- HSI_CONNECTION: thin dotted (#8B6FE8), lineWidth 0.8
- REVERSE_SALIENT: dashed orange (#FF8C42), lineWidth 1.2
- Default (BELONGS_TO, SEGMENT_OF, etc.): very thin gray, 0.3 alpha
- Arrowheads: small triangle (8px) drawn at the edge endpoint, rotated to match edge angle
- When an edge is connected to the hovered node, brighten it (double lineWidth, higher alpha)

**highlightCluster API (per GRAPH-06):**
- `graph.highlightCluster(group)` -- `group` can be a section name (e.g., 'market-analysis') or edge type (e.g., 'CONTRADICTS')
- If section name: find all nodes where `data.section === group` or `data.id === group`, temporarily increase their radius by 6 and set full opacity. Everything else dims. Reset after 2 seconds.
- If edge type: find all edges where `data.type === group`, highlight those edges and their connected nodes. Reset after 2 seconds.
- Expose on the instance: `const graph = new CanvasGraph(el, data); graph.highlightCluster('market-analysis');`

**Click handler (per GRAPH-07):**
- On canvas click, if a node is hovered, call `options.onNodeClick(nodeData)` where nodeData is the full node.data object from graph.json
- If no onNodeClick provided, default to calling `GraphDetailPanel.show(nodeData, edges)` if GraphDetailPanel is available on window

**Public API:**
- `new CanvasGraph(container, graphData, options)` -- constructor, starts animation
- `graph.highlightCluster(group)` -- programmatic highlight
- `graph.destroy()` -- stop animation, remove canvas, clean up listeners
- `graph.resize()` -- manually trigger resize (also handled by ResizeObserver)

**Module pattern:**
- Assign `window.CanvasGraph = CanvasGraph` for script-tag usage
- Also support `if (typeof module !== 'undefined') module.exports = CanvasGraph` for CJS
