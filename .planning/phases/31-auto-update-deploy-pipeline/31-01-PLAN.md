  <action>
Create `lib/presentation/presentation-watcher.cjs` -- follows the exact same pattern as `lib/wiki/wiki-watcher.cjs` but watches the room directory (not just .md files). Key differences from wiki watcher:
- Watch the entire room dir (same ignored patterns: .lazygraph, .reasoning, node_modules, .git)
- Trigger on ALL file types (not just .md) since presentation regeneration depends on graph.json, STATE.md, MINTO.md, assets
- awaitWriteFinish: stabilityThreshold 500ms (presentation generation takes ~2-3s, need stability)
- On change: (1) call `generate-presentation.cjs` via execSync/spawn to regenerate views, then (2) broadcast SSE reload event
- Debounce: 1000ms debounce on the regeneration call to avoid hammering during rapid multi-file writes (post-write hook can touch several files quickly)
- Export: startWatcher(roomDir, outputDir, pluginRoot), addSSEClient(res), removeSSEClient(res)

Create `lib/presentation/presentation-server.cjs` -- lightweight Express static server:
- Serve `outputDir` (the exports/presentation/ folder) as static files with express.static
- SSE endpoint at `/__sse` -- uses addSSEClient from presentation-watcher
- Inject a small SSE client script into HTML responses: intercept .html requests, read the file, inject `<script>` before `</body>` that connects to `/__sse` and calls `location.reload()` on message
- The SSE script injection happens server-side (NOT baked into templates) so deployed/static versions have zero SSE code
- Start chokidar watcher on the ROOM directory (not the output dir) via presentation-watcher.cjs
- On watcher change: spawn `node generate-presentation.cjs roomDir --output outputDir` then broadcast reload
- Accept roomDir, port, pluginRoot as args. Default port 8422 (8421 is wiki)
- Export: startPresentationServer(roomDir, port)

Create `scripts/serve-presentation` -- Bash launcher (follow serve-wiki pattern exactly):
- Default port 8422, range 8422-8430
- Resolve ROOM_DIR from $1 or $ROOM_DIR env var or ./room
- Find plugin root via SCRIPT_DIR
- Port scanning via python3 socket bind test (same as serve-wiki)
- Start server, trap cleanup, open browser at http://localhost:PORT/
- Print status with port and PID
- Wait for server process

IMPORTANT: The SSE client script injected by the server must be minimal (~10 lines):
```javascript
(function(){var es=new EventSource('/__sse');es.onmessage=function(){location.reload()};es.onerror=function(){setTimeout(function(){location.reload()},3000)}})();
```
