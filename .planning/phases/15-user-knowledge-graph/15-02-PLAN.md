  <action>
Extend the existing `scripts/post-write` hook to trigger lazygraph indexing when a room artifact is written. Add AFTER the existing analytics tracking block but BEFORE the classify-insight exec:

```bash
# Index artifact in LazyGraph (KuzuDB) if in room/ and is a .md file
if [[ "$FILE_PATH" == *.md ]]; then
  # Skip STATE.md and ROOM.md (not artifacts)
  basename_file=$(basename "$FILE_PATH")
  if [[ "$basename_file" != "STATE.md" ]] && [[ "$basename_file" != "ROOM.md" ]]; then
    # Walk up from FILE_PATH looking for STATE.md to find room root
    room_dir=""
    check_dir=$(dirname "$FILE_PATH")
    while [ "$check_dir" != "/" ] && [ "$check_dir" != "." ]; do
      if [ -f "$check_dir/STATE.md" ]; then
        room_dir="$check_dir"
        break
      fi
      check_dir=$(dirname "$check_dir")
    done
    if [ -n "$room_dir" ] && [ -d "$room_dir" ]; then
      # Run graph index in background with timeout (don't block hook)
      PLUGIN_ROOT="$(cd "$(dirname "$0")/.." && pwd)"
      timeout 2 node "${PLUGIN_ROOT}/bin/mindrian-tools.cjs" graph index "$room_dir" "$FILE_PATH" --raw 2>/dev/null &
    fi
  fi
fi
```

Key design decisions:
- Run in background (`&`) so it does not block the 3s hook timeout
- Use `timeout 2` as a safety net (KuzuDB single-artifact index should take <100ms)
- Use `--raw` flag to suppress formatted output (this is a background process)
- Only trigger for `.md` files, skip STATE.md and ROOM.md
- Walk up from FILE_PATH looking for STATE.md to find room root — this is robust against paths containing "room" as a substring (e.g., "showroom/notes.md" would not false-match)

Do NOT change the final `exec bash "${SCRIPT_DIR}/classify-insight"` line — it must remain the last command.
