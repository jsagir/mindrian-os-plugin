1. Node types with all properties and their types
2. Relationship types with direction and properties
3. Example Cypher queries for common questions:
   - "What artifacts are in the market-analysis section?" -> `MATCH (a:Artifact)-[:BELONGS_TO]->(s:Section {name: 'market-analysis'}) RETURN a.title, a.id`
   - "What contradicts my pricing model?" -> `MATCH (a:Artifact)-[:CONTRADICTS]->(b:Artifact) WHERE b.title CONTAINS 'pricing' RETURN a.title, a.section`
   - "What informs the solution design?" -> `MATCH (a:Artifact)-[:INFORMS]->(b:Artifact) WHERE b.section = 'solution-design' RETURN a.title, a.section`
   - "Show all cross-section relationships" -> `MATCH (a:Artifact)-[r]->(b:Artifact) WHERE a.section <> b.section RETURN a.title, type(r), b.title`
   - "Which sections are most connected?" -> `MATCH (a:Artifact)-[:BELONGS_TO]->(s:Section) WITH s, count(a) AS artifacts RETURN s.name, artifacts ORDER BY artifacts DESC`
4. KuzuDB Cypher dialect notes (from research): no APOC, walk semantics (add upper bounds to variable-length paths), SHORTEST keyword, list_ prefix functions, schema-first requirement
5. Note that ENABLES and INVALIDATES edges require explicit frontmatter markers in Tier 1 (full detection is Tier 2)
