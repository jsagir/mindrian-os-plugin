**Frontmatter:**

**Routing:** Parse user input to determine subcommand. If no subcommand, default to `list`. Subcommands: list, new, open, close, archive, where.

**Subcommand: list** (`/mos:rooms` or `/mos:rooms list`)

1. Run `bash scripts/room-registry list` to get JSON array of rooms
2. If no registry exists yet, check for legacy `room/` directory:
   - If `room/` exists: suggest adopting it with `/mos:rooms new` or tell them they have a single-room workspace
   - If no room at all: suggest `/mos:rooms new` or `/mos:new-project`
3. Render using Body Shape B (Semantic Tree) per UI system:

```
-- MindrianOS -- Rooms -----------------------------------------------

  ▼ .rooms/
  |- ■ acme-robotics          active   Pre-Opportunity   8 entries
  |- ▶ fintech-startup        parked   Discovery         14 entries
  |- ▷ biotech-venture        archived Validation        22 entries

  Active: acme-robotics (switched 2 hours ago)

  ▶ /mos:rooms open fintech-startup   Switch to parked room
  ▷ /mos:rooms new                    Create a new room
  ▷ /mos:rooms where                  Quick sanity check
```

Symbols: `■` = active, `▶` = parked, `▷` = archived. Entry counts from counting .md files in the room directory (excluding STATE.md, ROOM.md, USER.md).

4. Zone 4 footer: suggest opening a parked room (if any), creating a new room, or running /mos:rooms where.

**Subcommand: new** (`/mos:rooms new <name>`)

1. Validate name: must be a valid directory slug (lowercase, alphanumeric, hyphens). If the user provides a human-readable name like "Acme Robotics", slugify it to "acme-robotics" and use the original as `venture_name`.
2. If `.rooms/registry.json` does not exist AND `room/` directory exists:
   - Ask user: "You have an existing room/ project. Want me to adopt it into the registry first?"
   - If yes: run `scripts/resolve-room $PWD --adopt` to create registry with existing room, then proceed to create the new room
   - If no: proceed (the old room/ will still work via legacy fallback)
3. Create directory at `rooms/<slug>/` with the 8 standard sections (same structure as new-project Step 4, but WITHOUT the onboarding conversation -- just empty ROOM.md files)
4. Run `bash scripts/room-registry create <slug> "rooms/<slug>" "<venture_name>" "Pre-Opportunity"`
5. The registry create command automatically sets the new room as active and parks the previous
6. Run `bash scripts/compute-state rooms/<slug> > rooms/<slug>/STATE.md`
7. Report success with Zone 1 header showing new room name, and Zone 4 footer suggesting /mos:new-project (for the full onboarding conversation) or /mos:rooms list

**Important:** `/mos:rooms new` creates the room STRUCTURE only. The deep exploration conversation happens when the user runs `/mos:new-project` afterward (or they can just start working). This keeps room creation fast.

**Subcommand: open** (`/mos:rooms open <name>`)

1. Validate room exists in registry. If not, show 3-line error:
   ```
   x Room not found: <name>
     Why: No room named "<name>" in .rooms/registry.json
     Fix: /mos:rooms list
   ```
2. Check room status. If archived, warn: "This room is archived. Opening it will set status to active. Continue?" Wait for confirmation.
3. Run `bash scripts/room-registry set-active <name>` -- this parks the previous active room and sets the new one.
4. Show success message with Zone 1 header now showing the switched room name.
5. Zone 4 footer: suggest /mos:status, /mos:room, /mos:rooms where

**Subcommand: close** (`/mos:rooms close`)

1. Get current active room via `bash scripts/room-registry get-active`
2. If no active room, show error: "No active room to close."
3. Run `bash scripts/room-registry update <active-name> status parked`
4. Clear the active field: run python3 to set `active` to empty string in registry
5. Show confirmation: "Room <name> is now parked. No active room."
6. Zone 4: suggest /mos:rooms list, /mos:rooms open <name>

**Subcommand: archive** (`/mos:rooms archive <name>`)

1. Validate room exists. Show 3-line error if not.
2. If room is currently active, warn: "This will park the active room and archive it. Continue?"
3. Run `bash scripts/room-registry archive <name>`
4. Show confirmation.
5. Zone 4: suggest /mos:rooms list, /mos:rooms open (if other rooms available)

**Subcommand: where** (`/mos:rooms where`)

1. Get active room via `bash scripts/room-registry get-active`
2. If no active room, say "No active room. Run /mos:rooms list to see available rooms."
3. If active room exists, read its registry entry and show:
   ```
   Active room: <name>
   Path: <absolute-path>
   Venture: <venture_name>
   Stage: <venture_stage>
   Last opened: <timestamp>
   ```
4. Zone 4: suggest /mos:status, /mos:rooms open, /mos:rooms list
