**skills/room-passive/SKILL.md -- Room Lock Guard (per ROOM-03):**

Add a new section after "## Filing Intelligence" titled "## Active Room Lock (Multi-Room Safety)":

```markdown
## Active Room Lock (Multi-Room Safety)

When `.rooms/registry.json` exists in the workspace, multi-room mode is active. The following rules apply to ALL file-writing operations:

### Before Filing ANY Artifact

1. **Resolve the active room**: Read `.rooms/registry.json` and check the `active` field
2. **Verify target path**: The file being written MUST be inside the active room's directory
3. **Block cross-room writes**: If the target path is inside a DIFFERENT room (e.g., writing to `rooms/fintech-startup/` when `rooms/acme-robotics/` is active), STOP and show:
   ```
   x Cannot write to inactive room: fintech-startup
     Why: Active room is acme-robotics -- writing to other rooms risks cross-contamination
     Fix: /mos:rooms open fintech-startup
   ```
4. **Allow non-room writes**: Files outside any room directory (e.g., workspace root, `.rooms/`, `00_Context/`) are always allowed
5. **Allow admin commands**: `/mos:admin` operations are room-agnostic and never blocked by the room lock

### Single-Room Mode (No Registry)

When no `.rooms/registry.json` exists, skip the active room check entirely. All writes to `room/` are allowed (backward compatibility).

### Key Principle

The room lock is a PREVENTION mechanism, not a recovery mechanism. It is better to block a write and ask the user to switch rooms than to allow cross-contamination and try to fix it later.
```

**lib/core/room-ops.cjs -- Add resolveRoom function:**

Add a `resolveRoom(workDir)` function that does the same resolution as `scripts/resolve-room` but in pure Node.js (no execSync, no child process). This is for the Node.js tooling layer.

```javascript
/**
 * Resolve the active room directory path.
 * Strategy 1: Read .rooms/registry.json
 * Strategy 2: Fall back to room/ directory
 * @param {string} workDir - Workspace root directory
 * @returns {string|null} Absolute room path, or null if no room found
 */
function resolveRoom(workDir) {
  const resolved = path.resolve(workDir);
  const registryPath = path.join(resolved, '.rooms', 'registry.json');

  // Strategy 1: Registry
  if (fs.existsSync(registryPath)) {
    try {
      const reg = JSON.parse(fs.readFileSync(registryPath, 'utf8'));
      if (reg.active && reg.rooms && reg.rooms[reg.active]) {
        const roomPath = path.resolve(resolved, reg.rooms[reg.active].path);
        if (fs.existsSync(roomPath)) return roomPath;
      }
    } catch (e) {}
  }

  // Strategy 2: Legacy
  const legacy = path.join(resolved, 'room');
  if (fs.existsSync(legacy)) return legacy;

  return null;
}
```

Add `const fs = require('fs');` at the top if not already imported. Export `resolveRoom` alongside existing exports.

```bash
# Multi-room context (ROOM-05)
REGISTRY="${WORK_DIR}/.rooms/registry.json"
if [ -f "$REGISTRY" ]; then
  room_count=$(python3 -c "
import json
r = json.load(open('$REGISTRY'))
print(len(r.get('rooms', {})))
" 2>/dev/null || echo "0")

  if [ "$room_count" -gt 1 ]; then
    active_name=$(python3 -c "
import json
r = json.load(open('$REGISTRY'))
print(r.get('active', ''))
" 2>/dev/null || echo "")

    other_rooms=$(python3 -c "
import json, datetime
r = json.load(open('$REGISTRY'))
active = r.get('active', '')
lines = []
for name, room in r.get('rooms', {}).items():
    if name == active:
        continue
    status = room.get('status', 'parked')
    sym = '▶' if status == 'parked' else '▷'
    stage = room.get('venture_stage', '')
    display_name = room.get('venture_name', name)
    line = f'  ├── {sym} {display_name:<25s} {status:<10s} {stage}'
    lines.append(line)
if lines:
    lines[-1] = lines[-1].replace('├──', '└──')
print('\n'.join(lines[:5]))
if len(lines) > 5:
    print(f'  ...and {len(lines)-5} more (/mos:rooms list)')
" 2>/dev/null || echo "")

    if [ -n "$other_rooms" ]; then
      context="${context}\n\n  Other rooms:\n${other_rooms}\n\n  ▷ /mos:rooms                      Manage your rooms"
    fi
  fi
fi
```

1. Reads the registry to count rooms
