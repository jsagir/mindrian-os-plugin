**scripts/resolve-room** (the keystone -- per ROOM-01):

Create an executable bash script that resolves the active room's absolute path. Two-strategy resolution:

1. **Registry strategy**: If `$WORK_DIR/.rooms/registry.json` exists, read `active` field, look up that room's `path` in `rooms` map, resolve to absolute path, verify directory exists. Use `python3 -c "import json; ..."` one-liners for JSON reading (this is the existing pattern in session-start line 16).

2. **Legacy fallback**: If no registry but `$WORK_DIR/room/` directory exists, return that path.

3. **No room**: Exit 1 with empty stdout.

Arguments:
- `$1` -- workspace directory (defaults to `$PWD`)
- `--adopt` flag -- if legacy `room/` exists and no registry, create `.rooms/registry.json` adopting the existing room, then return its path. The adopted room gets: `path: "room"`, `status: "active"`, `venture_name` extracted from `room/STATE.md` if it exists (grep for `project_name:` line), `venture_stage` extracted similarly, `created` set to current timestamp, `last_opened` set to current timestamp.

Output: absolute path on stdout (exit 0) or empty (exit 1).

Performance: must complete in under 200ms. Single python3 invocation, no loops.

**scripts/room-registry** (CRUD operations -- per ROOM-01):

Create an executable bash script with subcommands:

- `create <name> <path> [venture_name] [venture_stage]` -- Add a room entry to registry. Create `.rooms/` dir and `registry.json` if they don't exist. Set new room as active, park any previously active room. Path is relative to workspace root.
- `read <name>` -- Print JSON for a single room entry.
- `list` -- Print all rooms as JSON array with name, path, status, venture_name, venture_stage, last_opened.
- `update <name> <field> <value>` -- Update a single field on a room entry (status, venture_name, venture_stage, last_opened).
- `set-active <name>` -- Set a room as active, park the previous active. Update last_opened timestamp.
- `archive <name>` -- Set room status to "archived". If it was active, clear the active field.
- `get-active` -- Print the active room name (or empty string if none).

All subcommands take `$WORK_DIR` as first argument (defaults to `$PWD`). Use `python3 -c "import json; ..."` for all JSON operations (consistent with codebase pattern). Write to a `.tmp` file first, then `mv` to `registry.json` (atomic write, prevents corruption).

Registry schema version: 1. Include `"version": 1` in all new registries.

```javascript
// Resolve room path: try registry first, fall back to room/
let roomDir = '';
const registryPath = path.join(dir, '.rooms', 'registry.json');
if (fs.existsSync(registryPath)) {
  try {
    const reg = JSON.parse(fs.readFileSync(registryPath, 'utf8'));
    if (reg.active && reg.rooms[reg.active]) {
      const resolved = path.resolve(dir, reg.rooms[reg.active].path);
      if (fs.existsSync(resolved)) roomDir = resolved;
    }
  } catch (e) {}
}
if (!roomDir) {
  const legacy = path.join(dir, 'room');
  if (fs.existsSync(legacy)) roomDir = legacy;
}
```
