**scripts/context-monitor:**
Create the statusline script that bridges context data to a temp file.

```
#!/usr/bin/env bash
# context-monitor: Statusline script for MindrianOS
# Reads context window data from Claude's statusline API
# Writes bridge file for skills/hooks to consume
# Runs after each assistant message (debounced ~300ms)
```

- Reads JSON from stdin via `cat`
- Extracts with jq: `model.id`, `model.display_name`, `context_window.used_percentage`, `context_window.context_window_size`, `context_window.remaining_percentage`
- Writes to `/tmp/mindrian-context-state` as simple KEY=VALUE format:
  ```
  MODEL_ID=claude-opus-4-6
  MODEL_NAME=Opus 4.6
  CTX_PCT=45
  CTX_SIZE=1000000
  CTX_REMAINING=55
  TIMESTAMP=1711108800
  ```
- Displays color-coded statusline output to stderr (NOT stdout -- statusline uses stderr for display):
  - Green (< 50%): `[MOS] 45% ctx | Opus 4.6`
  - Yellow (50-70%): same but yellow ANSI
  - Orange (70-85%): same but orange
  - Red (> 85%): `[MOS] 87% ctx | Opus 4.6 -- consider /clear`
- Use integer comparison for thresholds: strip decimal with `${CTX_PCT%.*}` before comparing
- If jq is not available or input is malformed, write conservative defaults to bridge file and output `[MOS] ? | unknown model`
- Make executable: `chmod +x`

Read `/tmp/mindrian-context-state` if it exists. If the file is missing or older than 5 minutes (compare TIMESTAMP to current epoch), use conservative defaults: assume 200K context window, 50% usage, unknown model.

| Model Contains | Context Size | Strategy |
|----------------|-------------|----------|
| opus | 1,000,000 | Rich context: load full methodology references inline when relevant |
| sonnet | 200,000 | Lean context: thin skills only, summarize references instead of quoting |

| Usage | Action |
|-------|--------|
| < 50% | Normal operation. Load references freely. |
| 50-70% | If user requests heavy methodology, mention context is moderate. |

When context is constrained (above 60% on Sonnet, above 80% on Opus):
- Do NOT load full methodology references inline

**scripts/session-start:**
Read the existing session-start script first. Add context budget injection AFTER the proactive Room intelligence section (after line ~47 where proactive_output is appended) and BEFORE the closing Larry instruction line.

Insert this block:
```bash
# Read context bridge file if available
CTX_BUDGET=""
BRIDGE_FILE="/tmp/mindrian-context-state"
if [ -f "$BRIDGE_FILE" ]; then
  # Check staleness (> 5 min = stale)
  BRIDGE_AGE=$(( $(date +%s) - $(stat -c %Y "$BRIDGE_FILE" 2>/dev/null || echo 0) ))
  if [ "$BRIDGE_AGE" -lt 300 ]; then
    CTX_MODEL=$(grep "^MODEL_NAME=" "$BRIDGE_FILE" | cut -d= -f2-)
    CTX_PCT=$(grep "^CTX_PCT=" "$BRIDGE_FILE" | cut -d= -f2-)
    CTX_SIZE=$(grep "^CTX_SIZE=" "$BRIDGE_FILE" | cut -d= -f2-)
    if [ -n "$CTX_PCT" ]; then
      CTX_BUDGET="[Context Budget] Model: ${CTX_MODEL:-unknown}, Usage: ${CTX_PCT}%, Window: ${CTX_SIZE:-200000} tokens"
      # Add warning if above threshold
      CTX_INT="${CTX_PCT%.*}"
      if [ "${CTX_INT:-0}" -ge 85 ]; then
        CTX_BUDGET="${CTX_BUDGET} -- CRITICAL: Consider suggesting /clear"
      elif [ "${CTX_INT:-0}" -ge 70 ]; then
        CTX_BUDGET="${CTX_BUDGET} -- WARNING: Context getting tight"
      fi
    fi
  fi
fi
```

Then append CTX_BUDGET to context string if non-empty:
```bash
if [ -n "$CTX_BUDGET" ]; then
  context="${context}\n\n${CTX_BUDGET}"
fi
```

Place this BEFORE the final "You are Larry..." line. The context budget info helps Larry adjust behavior from the very first message of a session.
