  <action>
Create 3 bash scripts and the project changelog:

**scripts/check-update:**
- Read current version from `"$PLUGIN_ROOT/.claude-plugin/plugin.json"` using `jq -r '.version'`
- Fetch CHANGELOG.md from `https://raw.githubusercontent.com/mindrian/mindrian-os-plugin/main/CHANGELOG.md` via `curl -sf`
- Extract latest version from first `## [X.Y.Z]` heading using grep
- Compare versions using `sort -V` (do NOT hand-roll semver parsing)
- Output format:
  - `UP_TO_DATE\nvX.Y.Z` if current == latest
  - `UPDATE_AVAILABLE\nCURRENT=X.Y.Z\nLATEST=X.Y.Z\n---CHANGELOG---\n{entries between versions}` if update available
  - `CHECK_FAILED\n{error message}` if curl fails
- PLUGIN_ROOT derived from `$(cd "$(dirname "$0")/.." && pwd)`
- Script must complete in under 3 seconds. No retries on failure.

**scripts/backup-modifications:**
- Takes optional arg: plugin root path (defaults to `$(cd "$(dirname "$0")/.." && pwd)`)
- Reads `.mindrian-checksums` from plugin root (file generated at install time with `md5sum` of each plugin file)
- If checksums file missing, output `NO_CHECKSUMS\nCannot detect modifications without install checksums.` and exit 0
- Compares current file checksums against stored checksums
- For each modified file, copies to `${WORK_DIR}/mindrian-patches/` preserving directory structure
- Creates `mindrian-patches/backup-meta.json` with: `{ "backup_date": "ISO8601", "plugin_version": "X.Y.Z", "modified_files": ["path1", "path2"] }`
- Output: `BACKED_UP\n{count} files backed up to mindrian-patches/` or `NO_MODIFICATIONS\nAll files match install checksums.`
- `WORK_DIR` is `$PWD` (user's project directory)

**scripts/reapply-modifications:**
- Reads `${WORK_DIR}/mindrian-patches/backup-meta.json`
- If no backup exists, output `NO_BACKUP\nNo backed-up modifications found.`
- Lists each backed-up file with its original path
- Output: `PATCHES_AVAILABLE\n{list of files}\nTo reapply, copy each file back to the plugin directory.`
- Does NOT auto-copy files back (plugin directory may be read-only cache). Provides clear instructions.

**CHANGELOG.md (project root):**
- Create initial changelog following Keep a Changelog format
- Single entry: `## [0.1.0] - 2026-03-22` with sections for Added (summarize phases 1-4 features: Larry personality, 26 methodology commands, pipeline chaining, proactive intelligence, Data Room dashboard, document generation, Brain MCP integration)
- Include `[Unreleased]` section at top (empty)
- This file is the source of truth for version history

All 3 scripts: `#!/usr/bin/env bash`, `set -euo pipefail`, executable permissions.

**commands/update.md:**
Create the `/mindrian-os:update` command following the thin command pattern.

Frontmatter:
```yaml
name: update
description: Check for MindrianOS updates, display changelog, backup your modifications
allowed-tools:
  - Bash
  - Read
  - Write
```

Command flow (Larry's voice throughout):

1. **Version Check**: Run `bash "${CLAUDE_PLUGIN_ROOT}/scripts/check-update"` and parse output
2. **If UP_TO_DATE**: Tell user their version in Larry's voice. "You're running the latest, v{version}. Nothing to update."
3. **If CHECK_FAILED**: Friendly error. "Couldn't reach the update server. Check your connection and try again."
4. **If UPDATE_AVAILABLE**: Display the changelog diff between current and latest version. Use Larry voice to highlight what matters.
5. **Offer Backup**: Ask user if they want to backup modifications first. If yes, run `bash "${CLAUDE_PLUGIN_ROOT}/scripts/backup-modifications"`
6. **Instruct Update**: Tell user to run: `claude plugin update mindrian-os@mindrian-marketplace` -- the command CANNOT reinstall itself (per research: "Plugin cannot reinstall itself; user must run /plugin update")
7. **Post-Update Reapply**: After update, suggest running `/mindrian-os:update reapply` to check for backed-up modifications

If user runs `/mindrian-os:update reapply`:
- Run `bash "${CLAUDE_PLUGIN_ROOT}/scripts/reapply-modifications"`
- If patches exist, list them and help user understand what to restore
- If no patches, confirm clean state

Voice rules: Conversational, not mechanical. Frame updates as "here's what's new" not "performing system update."
