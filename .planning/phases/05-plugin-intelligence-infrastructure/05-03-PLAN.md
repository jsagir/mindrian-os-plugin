Command flow (Larry's voice):

**Default behavior (`/mindrian-os:radar`):**
1. Read `references/capability-radar/capabilities-index.md`
2. Read `references/capability-radar/changelog-cache.md`
3. Present a summary in Larry's voice, organized by domain
4. Highlight capabilities most relevant to the user's current work (if Room exists, check STATE.md for venture stage)
5. If changelog cache is stale (> 7 days or "never"), suggest: "Run `/mindrian-os:radar --fetch` to check for new Claude capabilities."
6. End with: "These are the Claude features that power MindrianOS. When Anthropic ships something new, this is where I'll tell you about it."

**With `--fetch` flag:**
1. Use WebFetch to fetch `https://raw.githubusercontent.com/anthropics/claude-code/main/CHANGELOG.md` with prompt "Extract the 10 most recent changelog entries. For each entry, provide: version, date, and a summary of changes. Focus on features related to: plugins, MCP, hooks, statusline, context window, models, agents."
2. Analyze the fetched changelog for entries relevant to MindrianOS domains (models, code, desktop_cowork, plugins_mcp, visualization)
3. Write a summary to `references/capability-radar/changelog-cache.md` with format:
   ```markdown
   # Changelog Cache

   Last fetched: {ISO date}
   Source: https://raw.githubusercontent.com/anthropics/claude-code/main/CHANGELOG.md

   ## Recent Changes Relevant to MindrianOS

   ### {version} ({date})
   - **Domain:** {domain tag}
   - **Change:** {summary}
   - **MindrianOS impact:** {how this could enhance the plugin}
   ```
4. Present the findings to the user in Larry's voice
5. If any capability is new (not in capabilities-index.md), highlight it as an opportunity: "This is new since our last check. It could mean {impact}."

**With `--domain {domain}` flag:**
1. Read capabilities-index.md
2. Filter to requested domain section
3. Show only that domain's capabilities with deeper detail

Voice rules: Enthusiastic but grounded. Larry is excited about new tools but honest about what's aspirational vs practical. Never hype features that don't exist. Frame everything as "how does this help YOU build your venture" not abstract tech news.
