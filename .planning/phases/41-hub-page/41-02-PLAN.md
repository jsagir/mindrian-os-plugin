Add 3 new extraction functions and call them from scanRoom().

**1. extractInsights(graphEdges, graphNodes, roomDir, sections)** -- per D-10 to D-13, HUB-04
- Priority order per AD-6/D-11:
  1. CRITICAL red team findings -- parse RED-TEAM-REPORT.md for `### C{N}. {title}` under `## CRITICAL`
  2. CONTRADICTS edges -- from graphEdges where type === 'CONTRADICTS', use source+target labels
  3. VALIDATES edges -- from graphEdges where type === 'VALIDATES'
  4. MINTO governing thoughts -- scan each section for MINTO.md, extract first `## Governing Thought` or `## Key Insight` heading content
  5. CONVERGES 3+ sections -- edges where type === 'CONVERGES' connecting nodes in 3+ different sections
- Per D-12: always include at least 1 positive (VALIDATES) if one exists in the graph
- Per D-10: max 5 total
- Return array of {text, severity ('CRITICAL'|'HIGH'|'VALIDATED'|'INSIGHT'|'CONVERGES'), source}

**2. extractRedTeam(roomDir)** -- per D-14 to D-16, HUB-05
- Per D-14/AD-7: First check room/RED-TEAM-REPORT.md, then scan all section articles for `methodology: red-team` or `methodology: challenge-assumptions` in frontmatter
- Parse severity_summary from frontmatter (format: "4 CRITICAL, 5 HIGH, 4 MEDIUM, 2 LOW")
- If no frontmatter summary, count `### C{N}` headings for CRITICAL, `### H{N}` for HIGH, `### M{N}` for MEDIUM, `### L{N}` for LOW
- Per D-16: return null if no red team data found
- Return {critical: N, high: N, medium: N, low: N, source: filename} or null

**3. extractMethodologyArticles(sections, roomDir)** -- per D-17, D-18, HUB-06
- Per D-17: scan every article's frontmatter for `methodology:` field
- Re-read each article file, call parseFrontmatter, check for methodology key
- Per D-18: collect {title, methodology (badge label), section, filename}
- Return array sorted by section

Add to model return:
```js
insights: extractInsights(graphEdges, graphNodes, roomDir, sections),
redTeam: extractRedTeam(roomDir),
methodologyArticles: extractMethodologyArticles(sections, roomDir),
```
