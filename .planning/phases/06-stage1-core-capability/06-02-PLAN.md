**scripts/transcribe-audio**: Create a bash script that wraps the Modulate Velma API.

Input: Audio file path as $1 argument. Supported formats: .mp3, .m4a, .wav
Output: Timestamped, speaker-labeled text to stdout (one line per segment: `[MM:SS-MM:SS] Speaker_N: text`)
Also outputs structured JSON to stderr (for the file-meeting command to parse emotions/confidence).

Script flow:
1. Validate input file exists and has supported extension
2. Check for VELMA_API_KEY in environment or in project .mcp.json (parse with jq)
3. If no API key found, output error: "Velma API key not configured. Run /mindrian-os:setup transcription or set VELMA_API_KEY" and exit 1
4. Upload audio to Velma API via curl (multipart form data)
5. Parse JSON response with jq:
   - Extract segments with speaker_id, text, start_time, end_time
   - Extract emotion data (only strong signals: score > 0.7)
   - Format as timestamped speaker-labeled text for stdout
   - Output full JSON to a temp file and print path to stderr
6. Handle errors: API errors, network failures, unsupported format, file too large

Use `set -euo pipefail`. Include usage message with `--help` flag. Make executable (include shebang).

Note: The exact Velma API endpoint URL should be configurable via VELMA_API_URL env var with a sensible default. If the Velma API spec changes, only this script needs updating.

**tests/fixtures/sample-velma-response.json**: Create a realistic mock Velma API response JSON with:
- 8-10 segments from 3 speakers
- speaker_id fields (speaker_1, speaker_2, speaker_3)
- Timestamps (start_time, end_time as floats)
- Text content matching the Zoom fixture themes (venture/startup context)
- Emotions array per segment (most with low scores, 2-3 with strong signals > 0.7 like "skeptical", "enthusiastic", "frustrated")
- Realistic structure that matches what the transcribe-audio script expects to parse

**tests/test-velma-diarization.sh**: Bash test script. Does NOT call the live API. Instead:
1. Feeds tests/fixtures/sample-velma-response.json to the jq parsing logic from transcribe-audio
2. Validates: (a) speaker count matches expected, (b) timestamps are present and ordered, (c) emotion signals above 0.7 are extracted, (d) output format matches `[MM:SS-MM:SS] Speaker_N: text`
3. Exit 0 on pass, exit 1 on fail with description
Make executable.
