**Structure:**

## Purpose
This reference guides Larry through the post-filing cross-relationship scan. This scan is NOT an afterthought -- it is the core wicked problem management value. The venture is a nested system (Simon 1962). Changes in one subsystem cascade to others. This scan finds those cascades.

## The 5 Relationship Types

For each of the 5 types, provide:

### 1. INFORMS
- **What:** New artifact references or provides context for content in another section.
- **Detection heuristic:** Does the new artifact mention topics, entities, or claims that appear in other sections? Look for: same company names, same technologies, same market segments, same people mentioned across sections.
- **Example:** "Tyler's ionic liquid insight in solution-design INFORMS the equipment cost discussion in financial-model."
- **Larry prompt template:** "I notice {artifact} in {section} is relevant to your {other_section}. The connection: {explanation}. Want to add a cross-reference?"

### 2. CONTRADICTS
- **What:** New artifact conflicts with an existing claim in another section.
- **Detection heuristic:** Look for: conflicting numbers (cost estimates, timelines, market sizes), conflicting assessments (optimistic vs pessimistic on same topic), conflicting strategies (different approaches to same problem).
- **Example:** "Tyler's $50K equipment estimate contradicts Elena's $120K projection in business-model."
- **Larry prompt template:** "Heads up -- {speaker}'s {segment_type} about {topic} in {section} conflicts with what's in {other_section}: {existing_claim}. This needs resolution. Want to dig into it?"
- **IMPORTANT:** This is the highest-value finding. Always surface contradictions prominently.

### 3. CONVERGES
- **What:** Themes from this meeting appear in 3+ other sections, suggesting a convergence point.
- **Detection heuristic:** After all filings, identify recurring themes (topics mentioned by multiple speakers, topics that got filed to multiple sections). If a theme touches 3+ sections, it's a convergence signal.
- **Example:** "Green chemistry came up in solution-design, market-analysis, AND competitive-analysis. This is a convergence point -- your venture might be crystallizing around green chemistry as a differentiator."
- **Larry prompt template:** "I'm seeing convergence: {theme} appeared in {count} sections ({list}). This might be a core thesis forming. Want to explore it with /mindrian-os:build-thesis?"

### 4. INVALIDATES
- **What:** New artifact makes an existing assumption stale or invalid.
- **Detection heuristic:** Compare the `assumptions` field of newly filed artifacts against existing artifacts' assumptions. If a new insight directly addresses or undermines an existing assumption's claim, it's an invalidation.
- **Example:** "Lawrence's advice that the patent landscape has changed INVALIDATES the assumption in legal-ip that 'no prior art exists for ionic liquid synthesis.'"
- **Larry prompt template:** "Warning: {speaker}'s {segment_type} may invalidate an assumption in {section}: '{assumption_claim}'. If this assumption was wrong, it impacts: {impacts}. Review this?"

### 5. ENABLES
- **What:** New artifact unblocks or creates an opportunity in another section.
- **Detection heuristic:** Look for: new capabilities mentioned (technology, team skills, funding), resolved blockers, new connections or resources offered (mentor introductions, partnerships).
- **Example:** "Lawrence offering to connect with Dr. Chen ENABLES the team-execution gap of needing analytical chemistry expertise."
- **Larry prompt template:** "{speaker}'s {segment_type} about {topic} could unblock {gap_or_need} in {section}. Want to create an action item?"

## Scan Protocol

Larry should perform the scan in this order:
1. **CONTRADICTS first** -- highest value, needs immediate attention
2. **INVALIDATES second** -- assumption staleness is the #1 underserved outcome
3. **CONVERGES third** -- convergence signals help the user see the big picture
4. **ENABLES fourth** -- unblocked opportunities are actionable
5. **INFORMS last** -- cross-references are useful but lower urgency

For each finding, present it and ask the user: APPROVE (take action) / ACKNOWLEDGE (note but defer) / DISAGREE (capture the disagreement as data -- rejection is data).

## Wicked Problem Context

Include a reminder: "The venture is a wicked problem. There is no definitive formulation. Every insight changes the problem itself. The cross-relationship scan is how we detect those changes. The user who runs file-meeting is not just filing a transcript -- they are updating their model of a system that co-evolves with every decision."
