  <name>Task 1: render-pdf Python orchestrator script</name>
  <files>scripts/render-pdf</files>
  <action>
    Create scripts/render-pdf as an executable Python3 script (#!/usr/bin/env python3, chmod +x).

    The script has 4 main functions:

    1. collect_room_data(room_dir: Path) -> dict:
       - Walk all 8 section directories under room_dir
       - For each .md file (skip ROOM.md, STATE.md, any file starting with _):
         * Read file, parse with markdown2.markdown(text, extras=["metadata", "fenced-code-blocks", "tables"])
         * Extract metadata dict from result.metadata
         * Store {filename, metadata, html: str(result), title: metadata.get("title", stem)}
       - Group by section name
       - Return {"sections": sections_dict, "metadata": {"generated": datetime.now().strftime("%Y-%m-%d"), "venture_name": infer_from_room(room_dir), "doc_type": doc_type}}
       - infer_from_room: read room/STATE.md if it exists and extract venture name, else "Untitled Venture"

    2. render_document(doc_type: str, room_data: dict, output_path: Path):
       - Map doc_type to template filename via DOC_TYPES dict (thesis->investment-thesis.html, etc.)
       - Create Jinja2 Environment with FileSystemLoader(str(TEMPLATES_DIR))
       - Load template by name
       - Render template with: sections, metadata, colors=SECTION_COLORS, labels=SECTION_LABELS, section_order=SECTIONS, doc_title (type-specific title like "Investment Thesis" or "Due Diligence Report")
       - Create FontConfiguration()
       - Load CSS from TEMPLATES_DIR / "destijl-base.css" with font_config
       - Render PDF via HTML(string=html_string, base_url=str(PLUGIN_DIR)).write_pdf(target=str(output_path), stylesheets=[css], font_config=font_config)
       - CRITICAL: base_url=str(PLUGIN_DIR) so @font-face url('assets/fonts/...') resolves correctly

    3. add_toc_bookmarks(pdf_path: Path, sections: dict):
       - Only for doc types with multiple sections (thesis, report) -- skip for summary, profile
       - Open PDF with fitz.open(str(pdf_path))
       - Build TOC entries: for each non-empty section, add [1, label, page_number]
       - To find page numbers: search each page for section label text using page.search_for()
       - doc.set_toc(toc), doc.save(str(pdf_path), incremental=True, encryption=0), doc.close()

    4. open_pdf(path: Path):
       - Detect platform: Darwin -> "open", Linux -> check WSL via /proc/version -> "wslview" or "xdg-open", Windows -> "start"
       - subprocess.run([opener, str(path)])
       - Wrap in try/except -- don't crash if opener fails

    CLI interface (argparse):
       python3 scripts/render-pdf <doc-type> [--room DIR] [--output PATH] [--no-open]
       - doc-type: required positional (thesis|summary|report|profile)
       - --room: default ./room
       - --output: default ./room/exports/{doc-type}-{date}.pdf
       - --no-open: skip opening PDF after generation

    Constants at module level:
       PLUGIN_DIR = Path(__file__).resolve().parent.parent
       TEMPLATES_DIR = PLUGIN_DIR / "templates"
       SECTIONS = ["problem-definition", "market-analysis", "solution-design", "business-model", "competitive-analysis", "team-execution", "legal-ip", "financial-model"]
       SECTION_COLORS = {8 mappings from reference doc}
       SECTION_LABELS = {8 mappings from reference doc}
       DOC_TYPES = {"thesis": "investment-thesis.html", "summary": "executive-summary.html", "report": "due-diligence.html", "profile": "profile.html"}

    Main flow:
       1. Parse args
       2. Validate doc_type in DOC_TYPES
       3. Ensure output directory exists (mkdir -p room/exports/)
       4. collect_room_data
       5. render_document
       6. If thesis or report: add_toc_bookmarks
       7. Print success message with output path
       8. If not --no-open: open_pdf

    Error handling:
       - If room_dir doesn't exist: print "No Data Room found. Run /mindrian-os:new-project first." and exit 1
       - If no room content in any section: print "Data Room is empty. Run some methodologies first." and exit 1
       - If template not found: print helpful message with available types and exit 1
       - WeasyPrint errors: catch and print with suggestion to check fonts
