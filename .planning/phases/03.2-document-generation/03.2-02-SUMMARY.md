## Decisions Made

- base_url set to PLUGIN_DIR (not TEMPLATES_DIR) so WeasyPrint resolves @font-face url('assets/fonts/...') correctly relative to plugin root
- DOC_TYPES dict only includes 4 types with existing templates -- no phantom entries for deck/brief that would cause template-not-found errors
- TOC bookmarks applied only to thesis and report (multi-section narrative docs); summary and profile are too short to benefit
- WSL browser detection uses /proc/version check with cmd.exe /c start fallback, matching serve-dashboard pattern
