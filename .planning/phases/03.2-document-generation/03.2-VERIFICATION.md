## Gaps Summary

**One blocker gap identified:** The `render_document()` function in `scripts/render-pdf` loads the CSS stylesheet via `CSS(filename=str(TEMPLATES_DIR / "destijl-base.css"))` without passing `base_url`. WeasyPrint resolves relative URLs in a CSS file from that file's own directory (`templates/`). The @font-face declarations use `url('assets/fonts/BebasNeue-Regular.ttf')` — which WeasyPrint tries to find at `templates/assets/fonts/` (does not exist). It silently falls back to DejaVu Sans.

The HTML rendering call on the same line correctly sets `base_url=str(PLUGIN_DIR)`, but that does not apply to the separately loaded CSS object.

**The fix is one line:** change
```python
css = CSS(
    filename=str(TEMPLATES_DIR / "destijl-base.css"),
    font_config=font_config,
)
```
to:
```python
css = CSS(
    filename=str(TEMPLATES_DIR / "destijl-base.css"),
    base_url=str(PLUGIN_DIR),
    font_config=font_config,
)
```

This was confirmed to fix font embedding: after adding `base_url=str(PLUGIN_DIR)`, PDFs embed `Bebas-Neue` and `Inter` instead of `DejaVu-Sans`. The font TTF files are real binaries, correctly placed at `assets/fonts/`, and the @font-face paths are correct — only the resolution base is wrong.
