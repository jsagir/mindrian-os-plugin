  <name>Task 1: Build graph data pipeline scripts</name>
  <files>scripts/build-graph, scripts/serve-dashboard</files>
  <action>
Create two bash scripts following existing script patterns (see analyze-room, compute-state for style conventions: set -euo pipefail, ROOM_DIR parameter, comment sections with unicode box chars).

**scripts/build-graph:**
1. Accept ROOM_DIR parameter (default: ./room)
2. Accept output path parameter (default: ./dashboard/graph.json)
3. Scan room/*/*.md files (excluding ROOM.md, STATE.md) to build nodes:
   - Each artifact becomes a node: `{ data: { id: filename, label: title (from first # heading or filename), section: parent-dir-name, color: SECTION_COLORS[section], methodology: from-frontmatter, created: from-frontmatter } }`
   - Each of the 8 sections becomes a group node: `{ data: { id: section-name, label: SECTION DISPLAY NAME, color: SECTION_COLORS[section] }, classes: "section-group" }`
   - Artifact nodes get `parent: section-name` to nest inside group nodes
4. Build edges from three sources:
   - FEEDS_INTO: Parse `pipeline` and `pipeline_stage` frontmatter. Artifacts in the same pipeline with consecutive stages get a FEEDS_INTO edge (stage N -> stage N+1)
   - CONTRADICTS/CONVERGES: Run `bash scripts/analyze-room "$ROOM_DIR"` and parse CONTRADICT/CONVERGE lines. Create edges between the referenced sections' most recent artifacts
   - INFORMS: Scan artifact content for `[[section-name]]` cross-references (e.g., `[[market-analysis]]`). Create edge from artifact to referenced section group node
5. Build intelligence object for chat (Plan 02 will consume this):
   - Parse all GAP lines into `intelligence.gaps[]` array
   - Parse all CONVERGE lines into `intelligence.convergence[]` array
   - Parse all CONTRADICT lines into `intelligence.contradictions[]` array
   - Build `intelligence.summary`: total_artifacts, sections_active (count > 0), sections_empty, venture_stage (from STATE.md)
6. Output JSON to the specified path: `{ "elements": { "nodes": [...], "edges": [...] }, "intelligence": {...} }`
7. Use `mkdir -p` to ensure dashboard/ directory exists before writing
8. Print "Generated graph.json: {N} nodes, {M} edges" to stdout

Use simple grep/sed for YAML frontmatter extraction (match only known fields: methodology, room_section, pipeline, pipeline_stage, pipeline_input, created). Do NOT write a full YAML parser. Use the extract_frontmatter / get_field pattern from the research doc.

Handle empty room gracefully: output 8 section group nodes with zero artifact nodes, empty intelligence arrays, and summary showing 0 artifacts.

**scripts/serve-dashboard:**
1. Accept optional PORT parameter (default: find available starting at 8420)
2. Run `bash scripts/build-graph "$ROOM_DIR"` first (ROOM_DIR from env or default ./room)
3. Find available port: start at 8420, use `python3 -c "import socket; s=socket.socket(); s.bind(('',PORT)); s.close()"` to test, increment on failure up to 8430
4. Start `python3 -m http.server $PORT --directory dashboard` in background (capture PID)
5. Open browser using cross-platform detection:
   - WSL: `grep -qi microsoft /proc/version 2>/dev/null` -> `cmd.exe /c start "" "http://localhost:$PORT"`
   - macOS: `open "http://localhost:$PORT"`
   - Linux: `xdg-open "http://localhost:$PORT"`
   - Fallback: print URL
6. Print "Dashboard running at http://localhost:$PORT (PID: $PID)"
7. Print "Press Ctrl+C to stop"
8. Trap EXIT/INT/TERM to kill the background Python process
9. Wait for the background process (so script stays alive until user kills it)

Make both scripts executable (chmod +x).
