# Phase 38 Plan 01: End-to-End Validation

## Objective

Validate the complete v5.0 Data Room Presentation System by running syntax checks on all scripts, verifying template integrity, testing the presentation generator on a synthetic room, and producing a comprehensive validation report with pass/fail for each checkpoint.

## Context

- @38-CONTEXT.md

## Tasks

### Task 1: Script Syntax Validation
type: auto

**Actions:**
1. Run `node --check` on all .cjs files in scripts/ and lib/core/
2. Run `bash -n` on all bash scripts in scripts/
3. Verify all template HTML files exist in templates/presentation/
4. Check that plugin.json, settings.json, hooks/hooks.json are valid JSON
5. Record pass/fail for each check

**Done criteria:**
- All .cjs files pass syntax check
- All bash scripts pass syntax check
- All 6 templates exist
- All JSON configs are valid

### Task 2: Presentation Generator Integration Test
type: auto

**Actions:**
1. Create a temporary test room with STATE.md, MINTO.md, and sample sections
2. Run generate-presentation.cjs against the test room
3. Verify all 6 HTML files are produced
4. Verify branding contract: MindrianOS logo, "Built with MindrianOS" footer, Mondrian color bar
5. Verify dual theme support (dark default, light variant references)
6. Clean up temporary test room

**Done criteria:**
- Generator produces all 6 HTML files without errors
- Branding elements present in every generated file
- No runtime errors during generation

### Task 3: Validation Report
type: auto

**Actions:**
1. Compile all results into 38-VALIDATION-REPORT.md
2. Include pass/fail table for every checkpoint
3. Include summary statistics
4. Document any failures with details

**Done criteria:**
- 38-VALIDATION-REPORT.md exists with complete results
- Every checkpoint has a clear pass/fail status

## Verification

All tasks produce documented pass/fail results. The validation report is the primary output.
