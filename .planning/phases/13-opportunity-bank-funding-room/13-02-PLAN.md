  <action>
    Add to existing `lib/core/opportunity-ops.cjs` (from Plan 01):

    1. `buildGrantQuery(roomDir)` - Context-driven query generation per locked decision. Reads room STATE.md for venture_stage, domain keywords, geography. Reads problem-definition/ for domain context. Returns `{ keyword, fundingCategories, eligibilities, geography, ventureStage }`. If room has insufficient context (no problem-definition content), return `{ insufficient: true, reason: "..." }` so Larry can explain what's needed.

    2. `searchGrantsGov(query)` - POST to `https://api.grants.gov/v1/api/search2` using Node.js built-in fetch. Parameters: keyword, fundingCategories, oppStatuses: 'posted', rows: 25. Returns parsed JSON results array. Handle errors gracefully (return empty array + error message, never throw). Use 10-second timeout.

    3. `searchSimplerGrants(query)` - POST to `https://api.simpler.grants.gov/v1/opportunities/search` using built-in fetch. Use query keyword (max 100 chars), opportunity_status posted, pagination page_size 25 sorted by relevancy. Returns parsed results. Same error handling as searchGrantsGov.

    4. `scanOpportunities(roomDir)` - Orchestrate context-driven scan: call buildGrantQuery, then call both APIs (Promise.allSettled for resilience), merge and deduplicate results, compute relevance scores. Returns `{ query_context, results: [{title, funder, program, amount, deadline, source, relevance_score, relevance_reasoning}], api_errors: [] }`. This is the confirm-first presentation data -- NOT auto-filed.

    5. `fileOpportunity(roomDir, opportunityData)` - Create a dated artifact file in room/opportunity-bank/ following opportunity-template.md schema. Filename: `YYYY-MM-DD-{slug}.md`. Create opportunity-bank/ directory if needed. Returns `{ filed: true, path: "..." }`.

    6. `rejectOpportunity(roomDir, opportunityData, reason)` - Capture rejection as data per CLAUDE.md ("rejection is data"). Write a rejection record to opportunity-bank/STATE.md (append to rejections list). Returns `{ rejected: true, reason }`.

    Create `references/opportunities/grant-api-patterns.md` documenting:
    - How room context maps to API query parameters
    - Domain-to-fundingCategory mapping (ST for science/tech, HL for health, etc.)
    - Geography-to-eligibility mapping
    - Relevance scoring approach (multi-factor: domain fit, stage match, eligibility, amount range)
