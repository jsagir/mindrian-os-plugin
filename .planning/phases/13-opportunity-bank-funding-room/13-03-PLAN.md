  <name>Task 1: Funding lifecycle operations + state computation</name>
  <files>
    lib/core/opportunity-ops.cjs,

  <action>
    Add to existing `lib/core/opportunity-ops.cjs`:

    1. `createFunding(roomDir, opportunitySlug, sourceOpportunityPath)` - Create per-opportunity folder at `room/funding/{slug}/`. Generate:
       - `STATUS.md` with frontmatter: stage: discovered, outcome: null, source_opportunity: `[[opportunity-bank/{source-filename}]]` (wikilink for FUND-04 cross-reference), deadline (from source opportunity), last_updated: today, transition_history: [{stage: 'discovered', date: today, note: 'Created from opportunity scan'}].
       - `metadata.yaml` with structured data from the source opportunity (funder, program, amount, deadline, eligibility, source_url).
       Create funding/ directory if needed. Returns `{ created: true, path, slug }`.

    2. `updateFundingStage(roomDir, slug, newStage, note)` - Read funding/{slug}/STATUS.md, validate stage transition (must follow Discovered > Researched > Applying > Submitted order -- no skipping, no backward). Update stage in frontmatter, append to transition_history, update last_updated. Write back. Returns `{ updated: true, previousStage, newStage }`. If invalid transition, return `{ updated: false, error: "Cannot transition from X to Y" }`.

    3. `setFundingOutcome(roomDir, slug, outcome)` - Set outcome attribute (awarded | rejected | withdrawn) on STATUS.md. Outcome is separate from stage per CONTEXT.md decision. Can be set at any stage after Submitted (or at any stage for withdrawn). Returns `{ set: true, outcome }`.

    4. `computeFundingState(roomDir)` - Scan all funding/{slug}/STATUS.md entries. Aggregate into a pipeline summary: count by stage, upcoming deadlines (sorted), entries needing attention (stale > 14 days without update). Write to `room/funding/STATE.md`. Returns the aggregated data.

    5. `computeOpportunityBankState(roomDir)` - Scan all opportunity-bank/*.md files. Aggregate into a summary: total opportunities, count by status (discovered/filed/promoted/rejected), recent additions, top by relevance. Write to `room/opportunity-bank/STATE.md`. Returns the aggregated data.

    Create `scripts/compute-opportunity-state` Bash script:
    - `#!/usr/bin/env bash` with `set -euo pipefail`
    - Takes roomDir as argument (default ./room)
    - Calls `node bin/mindrian-tools.cjs opportunity compute-state "$roomDir"`
    - This wraps the Node.js computation for pipeline integration with existing compute-state chain
    - Make executable (chmod +x)
  </action>

  </verify>
  <done>createFunding creates per-opportunity folder with STATUS.md containing wikilink cross-reference. updateFundingStage validates 4-stage transitions. computeFundingState aggregates pipeline into STATE.md. compute-opportunity-state script wraps the computation.</done>
