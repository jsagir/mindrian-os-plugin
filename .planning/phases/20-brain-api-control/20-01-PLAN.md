Create directory `mcp-server-brain/sql/` and file `01-brain-api-keys.sql` containing:

1. **brain_api_keys table** (per D-02): `id` (uuid pk, default gen_random_uuid()), `api_key` (uuid, unique, NOT NULL), `email` (text, NOT NULL), `name` (text, NOT NULL, default ''), `plan` (text, NOT NULL, default 'free', CHECK plan IN ('free', 'pro', 'admin')), `is_active` (boolean, NOT NULL, default true), `expires_at` (timestamptz, nullable — NULL means permanent per D-03), `created_at` (timestamptz, NOT NULL, default now()), `last_used_at` (timestamptz), `request_count` (integer, NOT NULL, default 0), `created_by` (text, NOT NULL, default 'jonathan').

2. **validate_brain_key(key uuid) RPC function** — LANGUAGE plpgsql, SECURITY DEFINER. Must do TWO things atomically:
   - UPDATE brain_api_keys SET last_used_at = now(), request_count = request_count + 1 WHERE api_key = key AND is_active = true AND (expires_at IS NULL OR expires_at > now())
   - RETURN QUERY SELECT b.plan::text, b.email::text FROM brain_api_keys b WHERE b.api_key = key AND b.is_active = true AND (expires_at IS NULL OR expires_at > now())
   CRITICAL: Must use `RETURNS TABLE(plan text, email text)` — auth.cjs expects an array of {plan, email} rows. Using RETURNS RECORD or SETOF will break the response shape.

3. **brain_access_requests table** (per D-10): `id` (uuid pk, default gen_random_uuid()), `email` (text, NOT NULL), `name` (text, NOT NULL, default ''), `message` (text), `created_at` (timestamptz, NOT NULL, default now()), `reviewed` (boolean, NOT NULL, default false).
